## ⭐ 트리거 규칙

### T-00. 분류 인자 누락 시 (필수 차단)
사용자가 `엠군: 네이밍 / [상품id]`만 호출하고 분류 인자(제품명/브랜드명/블로그제목/상품명) 누락 시:
→ **묻기 (추정 X)**. "어느 분류의 네이밍이 필요하신가요? 제품명·브랜드명·블로그제목·상품명 중 선택."

### T-01. 02 미완 호출 시
02 포지셔닝 미완 상태에서 03 호출 시:
→ 차단: "02 포지셔닝·CV가 확정된 후 호출하셔야 합니다. CV 없이 이름 지으면 후속 카피·키워드 흔들립니다."

### T-02. 제품명·브랜드명 — KIPRIS 검색 권고 (필수)
제품명·브랜드명 후보 생성 후:
→ "KIPRIS (kipris.or.kr)에서 동일·유사 상표 검색 필수" 사용자에게 안내
→ 후보 표에 "KIPRIS 검색: 미확인" 표기
→ 사용자가 검색 후 결과 알려주면 후보 갱신

### T-03. 글로벌 확장 가능성 (브랜드명)
브랜드명 분류이면서 글로벌 확장 가능성 있는 제품:
→ 영문 발음·뜻 검토
→ "한글 ○○ → 영문 표기 ○○ → 영어권 발음·뜻 검토" 추가

### T-04. 블로그제목 — 05 키워드 일관성 (재호출 시 권고)
블로그제목 분류 시:
- **05 미완**: 02 CV 핵심 언어로 임시 작성 + "05 완료 후 키워드 일관성 재점검 권고" 명시
- **05 완료**: 정보·구매 키워드 분리 결과 활용 + 키워드별 1개씩 블로그제목 후보 생성

### T-05. 상품명 — 플랫폼 명시
상품명 분류 시:
→ 사용자에게 어느 플랫폼인지 묻기 (스마트스토어/쿠팡/11번가/카페24)
→ 플랫폼별 SEO 가이드 적용

### T-06. 시리즈 확장 검토 (브랜드명)
브랜드명 분류 시:
→ 시리즈 확장 가능성 검토 (단일 제품 X)
→ 확장 예 2~3개 같이 제시

### T-07. 상표 침해 우려 발견 시
대기업 상표·유명 브랜드와 유사한 후보 발견 시:
→ `1주차미피 06 07 대기업 상표 먹잇감_2clean` 정독
→ 상표 빈틈 공략 가능 여부 평가 (단순 유사 X, 적극 빈틈)

### T-08. 입에 달라붙기 검증
모든 후보:
→ 한국어 발음 3~5음절 권장
→ 영문 단어 포함 시 발음 자연스러움 확인
→ "고객이 입으로 부를 수 있는가" 셀프 테스트

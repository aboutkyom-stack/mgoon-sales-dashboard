## 0. GPT Image 2.0 핵심 변화 3가지 (프롬프팅에 영향을 주는 부분)

1. **네이티브 추론(Reasoning) 탑재** — 이미지 생성 전 레이아웃과 구도를 스스로 계획·검증. 복잡한 다중 레이아웃 작업에서 안정적인 결과가 나오는 이유
2. **한글 포함 다국어 텍스트 정확도 99%** — 이전 90~95%에서 거의 완벽 수준으로 향상
3. **단일 프롬프트로 최대 8개 이미지 동시 생성** — 일관된 스타일 유지

## 1. 카드 뉴스 만들기

### 1-1. 정보 수집 프롬프트
```
GPT 이미지 2 모델로 이미지 생성을 잘하기 위한 프롬프트 가이드를 
OpenAI의 공식 문서에서 찾아서 알려줘.
```
> 웹 검색으로 실제 공식 문서를 찾아 정리해줌

### 1-2. 카드 구성 정리 프롬프트
```
이 정보를 바탕으로 카드 뉴스를 만들고 싶어. 
각 카드에 담아야 할 정보를 정리해줘.
```
> 표지 + 카드 1~8번의 항목별 구성을 먼저 뽑고, 이후 이미지를 생성하면 완성도가 높아짐

### 1-3. 표지 이미지 생성 프롬프트 (스타일 지정 예시)
- **스타일 키워드**: `스위스 에디토리얼 캐러셀 스타일`
  - 격자 구조 + 굵은 타이포그래피 + 여백 중심의 미니멀 디자인
  - 정보 전달형 카드 뉴스에 특히 잘 어울림
- **이미지 비율**: 1:1 정사각형

> **핵심 팁**: 콘텐츠 구조를 먼저 텍스트로 잡아두고 → 각 항목별로 이미지를 순서대로 생성할 것

## 2. PPT 슬라이드 만들기

### 2-1. 슬라이드 구성 정리 프롬프트
```
(보고서 파일 첨부 후)
이 보고서를 바탕으로 PPT 슬라이드를 만들고 싶어.
각 슬라이드에 담아야 할 정보를 정리해줘.
```

### 2-2. 디자인 컨셉 제안 요청 프롬프트
```
이 슬라이드에 어울리는 디자인 컨셉을 몇 가지 제안해줘.
```
> 제안 예시: 맥킨지 전략 컨설팅 스타일 / 블룸버그 스타일 / 오픈AI 스타일 / 다크테크 인베스터덱 스타일 / SaaS 프로덕트 대시보드 스타일 등

- **이미지 비율**: 와이드스크린 16:9

### 2-3. 이미지 수정 방법 ① — JSON 형식 활용

**Step 1.** 수정할 슬라이드 이미지 첨부 후:
```
이 슬라이드의 모든 디자인 요소를 분석하여 JSON 형식으로 출력해줘.
```
> 텍스트, 색상, 레이아웃 정보가 키-값 구조로 출력됨

**Step 2.** 출력된 JSON을 새 창에 붙여넣고 원하는 값 직접 수정
- 예: 텍스트 변경, 하이라이트 컬러를 핑크로 교체 등

**Step 3.** 원본 슬라이드 이미지 + 수정한 JSON 파일을 함께 첨부 후:
```
이 JSON 형식대로 PPT 슬라이드를 만들어줘.
첨부한 이미지는 참고만 해라.
```

> **활용 상황**: 바꿀 값이 명확할 때 / 텍스트·색상 등 여러 요소를 한 번에 일괄 교체할 때

### 2-4. 이미지 수정 방법 ② — 영역 선택 드래그 수정
```
(수정할 영역을 드래그 선택 후)
이 텍스트를 "핵심 요약입니다"로 수정하고 컬러를 보라색으로 바꿔줘.
```
> 선택한 부분만 정확하게 수정됨. 단, 두 방법 모두 레이아웃 자체를 자유롭게 바꾸는 데는 한계가 있음

### 2-5. 레이아웃까지 수정하고 싶을 때 — Canva 매직 레이어
- 이미지를 다운로드 → Canva의 **Magic Layer** 기능 활용 (언어 설정 영문으로 변경 필요)
- 이미지 내 모든 요소가 수정 가능한 레이어로 변환됨
- 완성 후 업스케일 별도 필요

## 3. 상세 페이지(광고 소재) 만들기

### 3-1. 멀티 컨셉 상세 페이지 생성 프롬프트 구조
```
(제품 이미지 첨부)
네 가지 컨셉의 상세 페이지를 나란히 배치해서 생성해줘.
2026년 한식 배달 외식업 광고 트렌드를 반영한 폰트와 레이아웃을 적용해줘.
비율은 1:1.
[참고 카피 문구들도 함께 삽입]
```

### 3-2. 기존 상세 페이지 개선 프롬프트
```
더 효과적인 구도나 배치가 있다면 알아서 연출해주고,
모든 텍스트를 한글로 표현해줘.
```

### 3-3. 레퍼런스 기반 상세 페이지 제작 — JSON 템플릿 추출법

**Step 1.** 참고하고 싶은 레퍼런스 이미지 첨부 후:
```
이 이미지들의 디자인 요소를 분석해서 JSON 형식으로 출력해줘.
```
> 폰트, 컬러 시스템, 타이포그래피 위계, 레이아웃 구조 등이 정리됨

**Step 2.** 내 제품 이미지 + JSON 파일 함께 첨부 후:
```
아래 JSON 파일을 참고해서 첨부한 이미지 속 제품의 상세 페이지를 만들어줘.
```
> 제품 정보가 실제와 다르게 들어갈 수 있으므로 JSON이나 프롬프트에서 직접 수정

## 4. 브랜드 키트 만들기

### 4-1. 브랜드 정보 사전 준비 항목 (컨텍스트가 구체적일수록 결과 품질이 높아짐)
- 브랜드 성격
- 타겟 유저
- 핵심 키워드
- 로고에서 전달해야 할 느낌
- 로고 방향
- 선호 스타일
- 피해야 할 스타일
- 컬러 톤

### 4-2. 브랜드 키트 생성 프롬프트
```
[브랜드 정보 전체 붙여넣기]
아래 브랜드 정보를 바탕으로 고퀄리티 멀티 페이지 브랜드 키트를 만들어줘.
```
> 로고 시스템, 타이포그래피, 컬러 팔레트, 비주얼 랭귀지 등 브랜드 아이덴티티 한 번에 생성

### 4-3. 멀티 생성 시 특정 페이지만 선명하게 뽑는 프롬프트
```
위 브랜드 키트에서 커버만 따로 생성해줘.
```
```
위 브랜드 키트에서 브랜드 오버뷰만 따로 생성해줘.
```
> 멀티로 생성하면 개별 화질이 아쉬울 수 있어 필요한 페이지를 따로 뽑는 것이 효과적

### 4-4. 로고 베리에이션 40개 동시 생성 프롬프트
```
위 이미지를 모티브로 하여 정사각형 로고 익스플로레이션 보드를 만들어줘.
40개의 로고 컨셉을 번호가 있는 모듈러 그리드 형태로 정리하여
컨셉 시트에 담아줘.
[출력 형식 디테일 추가 가능]
```
> 개수가 많을수록 개별 로고 디테일이 떨어지므로 40개 내외 권장

### 4-5. 원하는 번호의 로고만 단독 추출
```
33번 로고를 단독으로 생성해줘.
```

### 4-6. 로고 배경 제거 (GPT Image 전용 기능)
```
이 로고 배경 제거해서 투명하게 만들어줘.
```
> Gemini/Imagen 계열에서는 미지원, GPT Image에서만 가능

### 4-7. 테이스트(TASTE) 스킬 활용법
- 브랜드 키트 스킬 MD 파일을 다운받아 GPT에 첨부
- 브랜드 아이덴티티 논리·시스템을 체계적으로 구성해줌 (로고 베리에이션, 컬러 팔레트, 타이포그래피, 텍스처, 아이콘 등)
```
(스킬 MD 파일 첨부 후)
첨부한 스킬을 사용해서 [브랜드명] 브랜드 키트를 만들어줘.
[브랜드 정보 함께 삽입]
```

## 5. UI 디자인 만들기

### 5-1. 웹 UI 랜딩 페이지 프롬프트 구조 예시
```
SaaS 생산성 도구의 랜딩 페이지 프론트엔드 디자인을 만들어줘.
구성 섹션: 히어로 / 기능 소개 / 후기 / CTA
디자인 방향:
- 딥 다크모드 팔레트 사용
- 강한 비대칭 구조의 히어로 섹션 적용
- 깔끔하고 시네마틱한 느낌
- 강렬한 이미지 사용
- 여유로운 레이아웃과 충분한 여백
```

### 5-2. UI 컴포넌트 라이브러리 일괄 생성
```
이 웹사이트 개발에 필요한 주요 컴포넌트 UI 보드를 생성해줘.
```
> 컬러 팔레트, 타이포그래피, 아이콘, 버튼, 인풋 필드 등 한 번에 생성
> 앞서 만든 웹사이트와 동일한 스타일·컬러 자동 유지

### 5-3. 섹션별 단독 추출
```
히어로 섹션만 따로 생성해줘.
```

### 5-4. 앱 UI 프롬프트 구조 예시
```
귀여운 3D 샌드위치 캐릭터가 대표 캐릭터로,
이 캐릭터가 페이지 곳곳에 들어가는 커스텀 푸드 주문 앱.
메인 컬러: 그린 + 화이트.
주요 키 스크린 3개를 생성해줘.
```

### 5-5. 유저 여정(User Journey) 시각화
```
이 앱의 사용자가 샌드위치를 커스텀하고 주문하기까지
어떤 여정을 거치게 되는지 프로세스를 보여줘.
```
> 메뉴 탐색 → 커스텀 → 재료 선택 → 소스 선택 → 미리보기 → 장바구니 → 결제까지 하나의 보드로 정리

### 5-6. 기존 Figma 시안 기반 새 화면 추가
```
(기존 앱 시안 이미지 5개 첨부 후)
앱의 온보딩 프로세스를 만들어줘.
```
> 기존 시안의 스타일·일러스트·컬러를 자동으로 이어받아 새 화면 생성

### 5-7. 테이스트 스킬 — 프론트엔드 웹 / 모바일 앱 버전
```
(프론트엔드 웹 스킬 또는 모바일 앱 스킬 MD 파일 첨부 후)
[요청 내용 작성]
```
> 브랜드 키트용 스킬과 별도로, 웹사이트·UI 이미지 퀄리티와 일관성을 높이는 데 특화

### 5-8. 브랜드 키트 → 웹사이트 연결 프롬프트
```
(앞서 만든 브랜드 키트 이미지 + 프론트엔드 웹 스킬 첨부 후)
온기반상 브랜드 키트 기반으로 웹사이트를 만들어줘.
```
> 브랜드 키트에서 잡은 컬러·로고·톤앤무드가 웹사이트 디자인에 그대로 이어짐

## 6. 공통 프롬프팅 원칙 정리

| 원칙 | 내용 |
|---|---|
| **구조 먼저** | 이미지 생성 전 텍스트로 콘텐츠 구조를 먼저 정리하고 시작 |
| **스타일 키워드 명시** | 스위스 에디토리얼, 맥킨지 스타일, 실리콘밸리 VC 피치덱 등 구체적인 스타일 용어 사용 |
| **브랜드 컨텍스트 사전 준비** | 성격·타겟·키워드·선호/비선호 스타일·컬러톤을 미리 정리해 프롬프트에 삽입 |
| **멀티 생성 후 단독 추출** | 전체 브랜드 키트 또는 여러 시안을 먼저 생성 → 마음에 드는 항목만 따로 고해상도로 재생성 |
| **JSON 형식 활용** | 디자인 요소를 JSON으로 추출 후 수정해서 재주입하면 일괄 수정 효율이 높아짐 |
| **레퍼런스 → JSON 템플릿화** | 마음에 드는 레퍼런스 이미지의 디자인 시스템을 JSON으로 추출 → 내 제품에 적용 |
| **스킬(TASTE) 활용** | 브랜드 키트, 프론트엔드 웹, 모바일 앱 전용 스킬 MD를 첨부하면 퀄리티와 일관성이 향상 |
| **비율 지정** | 카드 뉴스 1:1 / 슬라이드 16:9 / 세로 콘텐츠 9:16 등 용도에 맞는 비율 명시 |
| **참고 카피 문구 삽입** | 들어갔으면 하는 텍스트는 프롬프트 안에 직접 참고 텍스트로 넣어둘 것 |

# 부분엠군화 작업 인수인계 문서

> **용도**: 새 채팅 세션에서 맥락 없이 이 문서만 읽어도 작업 이어받기 가능.
> 강의 추가, 누락 발견, 재작업 시에도 이 문서를 기준으로 한다.
> **최종 업데이트**: 05_channel 완료 시점 (Phase 1 MD 파일 작성 전체 완료).

---

## 1. 이 프로젝트가 무엇인가

### 배경
엠군(M군)은 온라인 쇼핑몰 판매 전략 강사. 강의 내용 전체가 `persona/mgoon/core.md`에 증류되어 있고, 이 core.md를 AI에 로드해서 마케팅 전략을 짜는 구조로 이미 사용 중.

**문제**: core.md가 너무 크다(47KB+). 매번 전체를 로드하면 비효율적이고, 특정 업무(예: 타겟 분석)에서 나머지 내용(예: 채널 전략)은 노이즈가 된다.

**해결책**: "부분엠군" — 업무별로 쪼개진 특화 에이전트.
- 결핍·타겟 분석만 담당하는 M군
- 포지셔닝만 담당하는 M군
- 상세페이지만 담당하는 M군
- 등등

### 최종 목표
DB에 등록된 제품(R)을 넣으면 → 부분엠군들이 순서대로 이어받아 → 타겟 분석 → 포지셔닝 → 상세페이지 → 물길 전략까지 자동으로 뽑아내는 파이프라인 구축.

## 2. 핵심 설계 결정사항 (이유까지 기억해야 함)

### 결정 1: 에이전트별 파일 구성 (역할에 따라 다름)

| 파일 | 역할 | 필요 에이전트 |
|------|------|------------|
| `core.md` | 프레임워크, 개념, 판단 기준, 출력 형식 | 전체 필수 |
| `examples.md` | _2clean에서 추출한 실전 예시·멘트·카피 사례 | 전체 필수 |
| `expressions.md` | 어떻게 말하는가 — 어투·표현 패턴·카피 구조 | 텍스트 생성 에이전트만 (03, 04, 05) |
| `qa_checklist.md` | 결과물 검수 전용 확정 규칙 (검수 단계에서만 로드) | 전체 필수 |

**이유**: 분석 에이전트(01, 02)는 구조화된 데이터 출력이 목적이라 표현 패턴이 불필요. 텍스트 생성 에이전트(03 네이밍, 04 상세페이지, 05 채널)는 실제 카피·이름·글을 쓰므로 어투·패턴 파일이 필수.

### 결정 2: 증류(Distilled) 방식
- _2clean 원본 전부를 담지 않는다
- 핵심 프레임워크 + 대표 예시만 선별해서 core.md에 담는다
- **이유**: _2clean 전부 담으면 파일이 수백KB → 의미없음. 인덱싱만 하면 원본 파일 의존성 생김 → 깃허브에 올릴 때 문제. 증류하면 런타임에 원본 없어도 됨.

### 결정 3: 원본 페르소나 절대 수정 금지
- 원본: `C:\Users\kyum\Desktop\자동화 공장\persona\mgoon\`
- 부분엠군은 새 폴더 `agents/`에 만든다. 원본 참조는 해도 되지만 수정은 금지.

### 결정 4: 타겟별 독립 파이프라인
- 제품 하나에 타겟이 여러 개 나올 수 있음
- 타겟 하나 = 독립 파이프라인 하나 (포지셔닝부터 채널까지 타겟별로 분리)
- 사용자 확인 단계: 결핍·타겟 M군이 후보 목록 발굴 → 사용자가 실행 타겟과 순서 결정 → 이후 자동 진행

### 결정 5: 강의 파일 레이어 이해
강의 하나당 파일이 2종 존재 (raw _transcript.txt는 우리 작업에 불필요, 복사 안 함):
```
{파일명}_transcript_2clean_gemini-flash.txt  ← 정제 대본 (실전 멘트·카피 원문)
{파일명}_transcript_3summary_gemini-flash.md ← 핵심 요약 (구조 파악, 빠른 리뷰)
```
작업 순서: 3summary로 전체 파악 → 핵심 _2clean 선별 읽기 → 증류해서 md 작성

## 7. 에이전트 파일 작성 가이드라인

새 에이전트 작성 시 반드시 이 순서로:

### Step 1: 담당 강의 파일 전부 읽기
1. 해당 에이전트의 모든 _3summary 파일을 읽는다 (개념·흐름 파악)
2. 핵심이 되는 _2clean 파일을 2~4개 선별해서 읽는다 (실전 멘트·예시 수집)
   - 선별 기준: 해당 에이전트 주제의 핵심 예시가 나오는 강의
   - 예) 제로마케팅이면 `2주차 09` _2clean, 욕구깡패면 `2주차 12` _2clean

### Step 2: core.md 작성
- **목표 분량**: 8~12KB
- **포함 내용**:
  - 이 에이전트의 역할 정의 (1문단, 명확하게)
  - 핵심 개념 정의 (표 또는 계층 구조)
  - 작업 단계별 프레임워크 (단계 번호 붙이기)
  - 대표 예시 1~2개 (개념 이해용, 짧게)
  - 출력 형식 (다음 에이전트에게 넘길 데이터 구조)
  - 판단 기준 요약

### Step 3: examples.md 작성
- **목표 분량**: 8~12KB
- **포함 내용**:
  - _2clean에서 추출한 강사 실제 멘트·카피 (가능하면 원문 그대로)
  - 제품별 적용 사례 (타겟·결핍·접근법·카피 패키지)
  - 실패 사례도 포함 (왜 실패했는지 이유까지)
  - 실전 체크리스트 (마지막에)

### Step 4: expressions.md 작성 (03, 04, 05만 해당)
- **목표 분량**: 5~8KB
- **포함 내용**:
  - 이 에이전트가 생성하는 텍스트의 어투·톤 가이드
  - 반복되는 표현 패턴 (시그니처 훅, 설득 전개 구조)
  - 금지 표현 목록
  - 원본 mgoon의 `expressions.md`에서 해당 에이전트에 맞는 부분 발췌 + 특화 추가
- **참고**: 원본 `persona/mgoon/expressions.md`를 읽고 에이전트 특화 버전으로 재작성

### Step 5: qa_checklist.md 작성 (전 에이전트)
- **목표 분량**: 2~4KB (규칙이 쌓일수록 늘어남, 초기엔 적어도 됨)
- **포함 내용**:
  - 이 에이전트 결과물에서 반복 발생하는 실수 패턴만
  - 확정 규칙만 (판단 여지 없는 것)
  - 규칙마다 위반 예 / 통과 예 포함
  - 항목 추가 기준 명시 (2회 이상 반복, 재발 가능, 확정 가능)

### 공통 규칙
- 제목은 한국어 + 영어 병기 (예: `관여도 (Involvement)`)
- 표는 `| 헤더 | 내용 |` 형식으로
- 코드 블록(```) 안에 예시 대화·카피·캐릭터 정보 넣기
- 완성 후 파일 크기 확인 (`wc -c` 사용)

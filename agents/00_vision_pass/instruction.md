## 작업 절차

### 1) 선행 정독 (의무)

- `agents/_shared/data_contract.md` — 권위 필드 vs 시각설명 분리 규약 (특히 D-01·D-02)

이 단계는 사고 작업이 아니므로 강의 정독 의무 없음. 단 트리거 시 강의 _2clean 정독.

### 2) 입력 자료 로드

- 사용자가 첨부한 이미지 (1장 이상)
- 상품 테이블 권위 필드 (있으면 — 카테고리·모델명·제조사 등으로 주 피사체 특정 보조)

### 3) 3레이어 추출

`core.md`의 출력 형식 그대로:
1. 텍스트 레이어 (원문 인용)
2. 사실 레이어 (보이는 것만)
3. 해석 레이어 (근거 함께)

### 4) 자가 검수 (V-01~V-06)

`core.md` 자가 검수 항목 6개를 출력 끝에 첨부.

### 5) 다음 단계 안내

```
"00 비전패스 완료. 검토 후 진행할 부분 있으면 말씀, 없으면 '진행' 또는 '엠군: 결핍타겟 / [상품id]'로 호출하세요."
```

## ⭐ 트리거 규칙

### T-01. 박스 인쇄 가격·인증 라벨 식별 시
이미지에 박스 인쇄된 정가/할인가/세일 표시 또는 KC·CE 등 인증 라벨이 보이면:
- 텍스트 레이어에 원문 그대로 인용 (필수)
- 사실 레이어에 단언 X
- 해석 레이어에 "박스 인쇄 라벨 — 권위 필드와 별도 확인 필요" 명시

→ `data_contract.md` D-01·D-02 위반 방지.

### T-02. 주 피사체 모호할 때
이미지에 여러 제품이 동등하게 보이면:
- 사용자에게 어느 것이 주 제품인지 묻기 (추정 X)
- 상품 테이블 권위 필드의 카테고리·모델명으로 좁힐 수 있으면 해당 항목 기준으로 주 피사체 판정 후 결과 보고

### T-03. 텍스트 가독 불가
이미지 해상도·각도로 텍스트 일부 읽기 어려우면:
- 읽힌 부분만 텍스트 레이어에 인용
- 못 읽은 부분은 "[판독 불가]" 표기 + 사용자에게 추가 이미지 요청

### T-04. 사용 장면·환경 이미지 포함 시
제품 단독이 아닌 사용 장면·환경 이미지가 있으면:
- 해석 레이어 "타겟 인상"에 활용
- 함께 등장한 인물·소품은 별도 표기

### T-05. 패키징·언박싱 이미지 포함 시
- 사실 레이어에 별도 항목 "패키징"으로 정리
- 구성품 박스에 인쇄된 텍스트도 텍스트 레이어에 인용
- 해석 레이어 "상세페이지·섬네일 활용 포인트"에 사은품·언박싱 연출 단서 메모

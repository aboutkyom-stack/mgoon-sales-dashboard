# Spec Extract — 시각설명에서 스펙 필드 자동 추출

## 역할
시각설명 텍스트(Vision Pass 결과)를 읽고, 제품 스펙을 구조화된 JSON으로 추출한다.

## 핵심 원칙
1. **명시된 사실만** — 시각설명에 텍스트로 적혀있는 것만 추출. 추측·상상 금지.
2. **불확실하면 null** — 어렴풋이 보이거나 애매하면 빈 값(null) 반환. 잘못된 값보다 빈 값이 낫다.
3. **단위 변환** — 무게가 kg 표기면 1000 곱해 g 변환. cm 외 단위면 cm로 변환.
4. **필드 외 정보 무시** — 가격·재고 등은 추출 대상 아님 (별도 처리). 인증번호는 시각설명에 명시되어 있으면 추출.

## 추출 대상 필드

아래 필드만 추출한다. 각 필드는 명시된 사실이 없으면 `null` 반환:

{SPEC_FIELDS_GUIDE}

## 출력 형식 (반드시 JSON)

```json
{
  "카테고리": "...",
  "서브카테고리": null,
  "모델명": "JF-2024A",
  ...
}
```

- 모든 필드를 키로 포함 (값이 없어도 키 자체는 포함, 값을 null로).
- JSON만 출력. 마크다운 코드펜스 없이.
- 숫자 필드는 number 타입으로 (문자열 X).

---

## 추가: 기타 제품 특징 (`_특징_bullet`)

SPEC_FIELDS에 담기지 않은 정보 중 **구매 결정에 영향을 줄 수 있는 것**을 골라 짧은 bullet 리스트로 정리한다. 출력 JSON에 `_특징_bullet` 키를 추가하고 값은 문자열 배열로.

### 포함 기준 — 다음 중 하나에 해당하면 포함

- **사용 방식·호환성**: 배터리 규격, 충전 방식, 앱 연동, 전압·플러그 타입 등
- **안전·편의 기능**: 자동 절전, 방수 등급, 잠금 기능, 열차단 소재 등
- **구성품의 부가가치**: 한글 설명서 포함, 보증서, 전용 파우치·케이스 동봉 등
- **대상·상황 제약**: 실내 전용, 어른·아이 겸용, 특정 연령·체형 적합 등
- **기술·소재 특이점**: LED 색상 변경, 내열 온도, 방염 처리, 항균 코팅 등

### 제외 기준 — 다음에 해당하면 제외

- 이미 SPEC_FIELDS에 채워진 항목과 겹치는 것 (색상, 재질, 치수 등 반복 금지)
- 제품명·카테고리만 봐도 당연히 알 수 있는 것 (예: "장난감입니다", "주방에서 씁니다")
- 수치 근거 없는 막연한 표현 (예: "고급 소재", "뛰어난 내구성", "튼튼합니다")
- 가격·재고·배송 관련 정보

### 형식

- 각 bullet 30자 이내, 명사형 또는 짧은 문장.
- 없으면 빈 배열 `[]` 반환. 억지로 채우지 말 것.
- 예: `"_특징_bullet": ["AA건전지 2개 별도 구매", "30초 무조작 시 자동 절전", "조립식 구성"]`

## 1. 핵심 방향 전환 (이번 세션에서 확정)

### 1-1. 영문 `gpt_prompt` 폐기 → 한국어 구조화 브리프로 직행

**배경**: 기존 04-1 출력은 섹션마다 한국어 마크다운 본문 + 영문 `gpt_prompt` 필드를 별도로 생성했다.

**실험**: 동일 제품·섹션을 영문 단락(기존)과 한국어 구조화 브리프(신규)로 각각 GPT Image에 입력해 결과 비교.

**결론**: 한국어 구조화 브리프가 우위.
- GPT Image 2.0 한글 정확도 99%
- 영문 단락보다 한국어 구조화 필드(`background`, `composition`, `mood`, `text_elements`, `design_notes`)가 레이아웃 제어 품질이 높음
- 영문 필드는 이중 작업이고 토큰 낭비

**변경 사항**:
- `gpt_prompt` 필드 생성 금지 (`core.md`, `instruction.md`, `qa_checklist.md` 모두 반영)
- JSON 한국어 필드가 곧 이미지 생성 입력값 — UI에서 마크다운 카드로 렌더링, 사용자가 GPT Image에 복붙

---

### 1-2. "유튜버 방식" 실험 → 기각

**시도**: 특정 유튜버가 공유한 프롬프팅 방법("짧고 자연스럽게, 알아서 연출")을 적용.

**결과**: 절반 수준의 품질. 상세페이지는 레이아웃·카피 위치·배지 색 등 픽셀 단위 제어가 필요한데, 자유 연출 방식은 그 제어가 불가능.

**결론**: 구조화 브리프(current) 유지. 유튜버 방식은 일반 사진 생성에는 유리하지만 상세페이지용으로는 부적합.

---

### 1-3. JSON-only 출력 정책

**문제**: 기존 core.md가 "마크다운 본문 먼저, JSON 블록은 끝에"로 지시 → 섹션이 많으면 마크다운 쓰다가 max_tokens 소진 → JSON 잘림 → DB 저장 실패.

**해결**: JSON 블록만 출력하는 정책으로 전환.
- `core.md`, `instruction.md` 모두 "JSON 블록 단 하나, 부가 텍스트 없음"으로 명시
- max_tokens 16384 (JSON-only이므로 충분)

---

### 1-4. max_tokens 조정

| 시점 | 값 | 이유 |
|------|-----|------|
| 초기 | 32000 | 마크다운 + JSON 동시 출력 |
| 다른 채팅 수정 | 32000 (유지 또는 상향) | Opus 4.7 컨텍스트 혼동으로 잘못 설정 |
| 이번 세션 확정 | **16384** | JSON-only + gpt_prompt 제거 후 충분 |

`pipeline/auto_pipeline.py:339`, `pages/2_pipeline.py` 모두 16384로 맞춤.

## 2. UI 변경 — 복붙 블록 자동 생성

**문제**: 각 섹션 디렉션을 GPT Image에 넣으려면 디자인 시스템(전역 규칙) + 섹션 브리프를 수동으로 합쳐야 했다.

**해결**: `pages/2_pipeline.py`에 `_build_section_copy_block()` 함수 추가.
- 디자인 시스템 + 해당 섹션 브리프를 마크다운 블록으로 자동 조립
- 각 섹션 카드 끝에 "GPT Image 복붙용" 코드 블록으로 렌더링
- 사용자는 그 블록만 복사해서 GPT Image 대화에 붙여넣으면 됨

## 3. SKILL_gpt_image.md 생성 (보류 단계)

**목적**: GPT Image 세션에 첨부하는 마스터 시스템 프롬프트. 반복적으로 요청하게 되는 규칙을 모아두는 파일.

**현재 포함된 규칙**:
- 포토리얼리즘 / 50mm 렌즈 / 배경 아웃포커스
- 한글 텍스트 verbatim (깨짐·변형 금지)
- 캔버스 1024×1280px (16px 배수)
- 금지: 로고, 워터마크, 중국풍·일본풍 배경, 외국인 인물
- 이미지-텍스트 영역 분리 (그라데이션 오버레이 지양)
- 헤드카피 의미 단위 2줄 분리
- 신뢰/인증 배지 강조 색 (회색 배지 금지)
- 가격: 정가(취소선) + 판매가(강조) 세트

**운영 방침**: 아직 GPT Image 세션에 일괄 첨부는 하지 않음. 규칙이 더 쌓이면 운영 결정.

## 5. 롤백 관련

### 롤백이 필요한 경우
- 최신 결과물 CTA 섹션처럼 배경색·임팩트 퇴보가 일관되게 나타나면 롤백 고려
- 롤백 포인트: `gpt_prompt` 있던 버전 → 단, 토큰 소진·JSON 잘림 문제 재발 주의

### 롤백 없이 개선하는 방향 (권장)
CTA 섹션 퇴보는 디렉션 내용 문제일 가능성이 높음. 한국어 브리프 정책 자체를 버리기보다:
1. `instruction.md`에 CTA 섹션 배경색 강제 지시 추가 검토
2. `qa_checklist.md`에 "CTA 섹션 배경색 명시" 규칙 추가 검토

## 8. 후속 개편 (2026-05-09)

### 운영 방식 확정 — GPTs 지침 + 디자인 시스템 분리

**배경**: 디자인 시스템(전역 규칙)이 섹션 복붙 블록마다 반복되어 GPT Image가 매번 다시 읽음 → 섹션 디테일에 집중하지 못해 품질 저하 (예: CTA 흰 배경 퇴보).

**확정된 워크플로우**:
1. **GPTs 지침에 SKILL_gpt_image.md 등록** (사전 1회)
   - "지식"(Knowledge)이 아닌 **"지침"(Instructions)** 에 등록
   - 지식은 RAG 단편 검색이라 규칙 누락 위험
2. **세션 시작 시** (사용자가 한 번에 첨부):
   - 제품 실사 사진 3~5장
   - 디자인 시스템 블록 (1회)
   - 첫 섹션 브리프
3. **2번째 섹션부터**: 섹션 브리프만 입력
4. **한 세션 = 한 제품** 원칙

### 코드 변경

**`pages/2_pipeline.py`**:
- `_build_section_copy_block(ds, sec)` → `_build_section_copy_block(sec)` 시그니처 변경
  - 디자인 시스템 부분 제거 → 섹션 정보만 lean하게 출력
- 신설: `_build_design_system_copy_block(ds)` — 디자인 시스템 단독 블록
- `_render_image_direction_panel`: 디자인 시스템 복붙 블록을 화면 상단에 1회만 렌더링, 섹션 블록에는 미포함

**`agents/04_1_image_direction/SKILL_gpt_image.md`** (전면 개편):
- GPTs 지침 등록 안내 헤더 + 세션 워크플로우 명시
- "제품 사진 사용 (최우선 규칙)" 추가 — 첨부 사진 그대로 사용, 임의 생성·변형 금지
- "CTA·클로징 섹션 배경" 규칙 추가 (강조색 사용, 흰 배경 지양)
- "컨텍스트 관리" 섹션 추가 — 같은 세션 안 일관성, 부분 수정 원칙

**LLM 에이전트 파일** (`core.md`, `instruction.md`, `qa_checklist.md`): **변경 없음**.
디자인 시스템은 여전히 JSON에 출력해야 함 (DB 저장·UI 표시 + 복붙 블록 분리에 필요).

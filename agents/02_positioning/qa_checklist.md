# 02 포지셔닝 — 자가 검수 체크리스트

> **로드 시점**: 포지셔닝 출력 직후.

---

## R-01. 구매 CV 소비자 시각
구매 CV 5개가 소비자 시각으로 뽑혔는가. 판매자 시각의 스펙·기능 나열 X.

- 위반 예: "0.5mm 두께 / 알루미늄 소재 / 무게 200g"
- 통과 예: "휴대성 / 사용 편의성 / 디자인 / 가격 / 내구성"

---

## R-02. CV 유불리 판단 명시
각 CV에 내 제품의 유리/불리/비등이 명시됐는가.

---

## R-03. 포지셔닝 맵 축 — 선호도 언어
X·Y축이 선호도 언어로 적혔는가. "거리", "가격" 같은 중립 단어 X.

- 위반 예: "X축: 가격 / Y축: 기능"
- 통과 예: "X축: 가성비↑ / Y축: 기능 충실도↑"

---

## R-04. 포지셔닝 맵 두 축 겹침 X
X·Y축이 비슷한 개념으로 겹치지 않는가.

- 위반 예: X축 '품질', Y축 '성능' (겹침)
- 통과 예: X축 '가성비', Y축 '디자인 만족도' (독립)

---

## R-05. 두 개 까고 두 개 넣기 — 사실 기반
까야 할 2가지와 넣어야 할 2가지가 모두 사실 기반인가. 막연한 비방 X.

- 위반 예: "다른 거 다 별로예요"
- 통과 예: "브랜드값 50%, 1년 안 잔고장 빈번 / 우리는 KC 인증 + 1년 무상 보증"

---

## R-06. 가치더하기 개수 상한 (3개)
가치더하기가 3개 이하인가. 욕구에 닿지 않는 억지 항목 없는가.

- 위반 예: 가치더하기 6개 (CV 훼손)
- 통과 예: 모듈화 + 단점 장점화 2개

---

## R-07. 제품 본질 분류
제품 본질이 5분류 중 하나로 명시됐는가 + 주의사항 한 줄 있는가.

---

## R-08. 같은 군 차별화 (있을 때만)
같은 군 차별화 항목이 비전패스 사실 + 같은 군 대비 우월 지점 형식인가.

- 위반 예: "우리 제품은 좋아요" (사실 기반 X)
- 통과 예: "VP 사실: 무게 200g vs 같은 가격대 평균 380g → 같은 군 휴대성 1위"

---

## R-09. POSITIONING_JSON 블록
출력 맨 끝에 `---POSITIONING_JSON---` ... `---END_POSITIONING_JSON---` 블록이 정확한 마커로 박혀있고 스키마가 맞는가.

특히 다음 필드 확인:
- `rule_engine_inputs` 6개 모두 채워짐
- `rule_engine_flags` 4개 모두 채워짐 (true/false)
- `persuasion_method_candidates` 최소 1개
- `구매핵심가치_이동_필요 = true`면 `구매핵심가치_이동_근거` 채워짐
- **`category_objections` 1~3개 채워짐** (모두 모름이면 `source: "모름"` 명시 + 사용자 추가 입력 요청 메시지 함께)

---

## R-11. category_objections 발굴 완료 ⭐
04_a 핵심 반론 선제 처리 블록 입력으로 사용될 `category_objections` 필드가 발굴됐는가.

- 위반 예: `category_objections: []` (빈 배열)
- 위반 예: `source: "AI 카테고리 추론"`인데 objection이 일반적·표면적 ("이거 안전한가요?" 수준)
- 통과 예: 비둘기 퇴치제 → "비둘기는 학습력 빠른데 익숙해지면 효과 없는 거 아닌가?" + counter + source 명시
- 통과 예: 추론 불가 시 `source: "모름"` 명시 + 사용자 추가 입력 요청을 출력 본문에 박음

## R-10. 사조참치 함정 회피 (T-02 발동 시)
상징제 시장에 기능제로 진입하려는 경우, 사조 안심따개 실패 사례를 인지하고 진입 근거가 충분히 강한가.

- 위반 예: 동원·오뚜기 시장에 "우리는 더 좋은 따개 있어요"로 진입
- 통과 예: 진입 X 권고 또는 진입 시 날카로운 제로마케팅 명확히 설계

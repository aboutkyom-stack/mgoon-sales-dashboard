## 1. 핵심 방향 전환 (이번 세션에서 확정)

### 1-1. 영문 `gpt_prompt` 폐기 → 한국어 구조화 브리프로 직행

**배경**: 기존 04-1 출력은 섹션마다 한국어 마크다운 본문 + 영문 `gpt_prompt` 필드를 별도로 생성했다.

**실험**: 동일 제품·섹션을 영문 단락(기존)과 한국어 구조화 브리프(신규)로 각각 GPT Image에 입력해 결과 비교.

**결론**: 한국어 구조화 브리프가 우위.
- GPT Image 2.0 한글 정확도 99%
- 영문 단락보다 한국어 구조화 필드(`background`, `composition`, `mood`, `text_elements`, `design_notes`)가 레이아웃 제어 품질이 높음
- 영문 필드는 이중 작업이고 토큰 낭비

**변경 사항**:
- `gpt_prompt` 필드 생성 금지 (`core.md`, `instruction.md`, `qa_checklist.md` 모두 반영)
- JSON 한국어 필드가 곧 이미지 생성 입력값 — UI에서 마크다운 카드로 렌더링, 사용자가 GPT Image에 복붙

### 1-2. "유튜버 방식" 실험 → 기각

**시도**: 특정 유튜버가 공유한 프롬프팅 방법("짧고 자연스럽게, 알아서 연출")을 적용.

**결과**: 절반 수준의 품질. 상세페이지는 레이아웃·카피 위치·배지 색 등 픽셀 단위 제어가 필요한데, 자유 연출 방식은 그 제어가 불가능.

**결론**: 구조화 브리프(current) 유지. 유튜버 방식은 일반 사진 생성에는 유리하지만 상세페이지용으로는 부적합.

---

### 1-3. JSON-only 출력 정책

**문제**: 기존 core.md가 "마크다운 본문 먼저, JSON 블록은 끝에"로 지시 → 섹션이 많으면 마크다운 쓰다가 max_tokens 소진 → JSON 잘림 → DB 저장 실패.

**해결**: JSON 블록만 출력하는 정책으로 전환.
- `core.md`, `instruction.md` 모두 "JSON 블록 단 하나, 부가 텍스트 없음"으로 명시
- max_tokens 16384 (JSON-only이므로 충분)

### 1-4. max_tokens 조정

| 시점 | 값 | 이유 |
|------|-----|------|
| 초기 | 32000 | 마크다운 + JSON 동시 출력 |

## 3. SKILL_gpt_image.md 생성 (보류 단계)

**목적**: GPT Image 세션에 첨부하는 마스터 시스템 프롬프트. 반복적으로 요청하게 되는 규칙을 모아두는 파일.

**현재 포함된 규칙**:
- 포토리얼리즘 / 50mm 렌즈 / 배경 아웃포커스
- 한글 텍스트 verbatim (깨짐·변형 금지)
- 캔버스 1024×1280px (16px 배수)
- 금지: 로고, 워터마크, 중국풍·일본풍 배경, 외국인 인물
- 이미지-텍스트 영역 분리 (그라데이션 오버레이 지양)
- 헤드카피 의미 단위 2줄 분리
- 신뢰/인증 배지 강조 색 (회색 배지 금지)
- 가격: 정가(취소선) + 판매가(강조) 세트

**운영 방침**: 아직 GPT Image 세션에 일괄 첨부는 하지 않음. 규칙이 더 쌓이면 운영 결정.

## 5. 롤백 관련

### 롤백이 필요한 경우

## 9. 후속 개편 (2026-05-10 ~ 11)

> 8장의 "GPTs 지침 등록" 결정은 **보류** 상태로 변경됨. 9장이 최신 결정.

### 9-1. 캔버스 표준 변경 — 너비 1024 → 860 + 비율 세트 도입

**배경**: 멀티 채널 운영(스마트스토어·쿠팡·카페24·옥션·지마켓·도매꾹) 필요. 가장 좁은 권장값(스마트스토어 860px)에 맞추지 않으면, 큰 너비로 만든 후 리사이즈 시 텍스트가 작아져 모바일 가독성 손실.

**확정**:
- **너비 860px 고정**
- 비율 세트:
  | 비율 | 권장 px | 용도 |
  |------|---------|------|
  | 4:5 | 860×1075 | 기본 상세 컷 |
  | 2:3 | 860×1290 | 세로 길이가 핵심인 제품(타워크레인 등), 정보 밀도 높은 씬 |
  | 1:1 | 860×860 | 정방형 강조 배너 |

**세로 강조 카피 ↔ 비율 정합 규칙**: "○○보다 큽니다", "1.3미터입니다" 같은 세로 길이 강조 카피가 있는 섹션은 반드시 2:3 선택. 4:5에 압축하면 제품이 작아 보여 카피와 시각이 따로 논다.

**API 자동화 전환 시 보정값** (현재는 GPT 웹 복붙 운영): 860→856, 1075→1072, 1290→1296 (가로·세로 모두 16배수 제약). instruction.md/SKILL.md에 주석으로 명시.

### 9-2. 섹션 분할 허용

**배경**: 한 콘티 행에 카피·요소가 과부하인 경우(예: 후기 3개 한 컷, 기능 설명 5종 한 컷) 한 캔버스에 다 담으면 여백이 사라지고 비율이 깨짐.

**확정**: 한 콘티 행을 2개 섹션으로 분할 가능. 분할 시 `name`에 `-A`/`-B` 접미사로 출처 표시. 분할 기준 = `text_elements` 4개 초과 OR `composition`에 독립 영역이 2개 이상.

**qa_checklist R-04 완화**: "콘티 행 수 = sections 수" → "콘티 행 ≤ sections 수, 분할 시 -A/-B 표기" 로 변경.

### 9-3. 인물 등장 컷 배경 아웃포커스 — 조건부 규칙

**배경**: 인물 컷의 80%는 인물·제품이 주인공이지만, 매장 전경·공장 현장·사용 환경 묘사 등 배경 자체가 맥락 전달의 핵심인 씬은 아웃포커스를 적용하면 안 됨.

**확정**: 04-1이 섹션 내용 보고 자동 판단해 `design_notes`에 `얕은 심도, 배경 아웃포커스`를 명시(또는 명시하지 않음).

### 9-4. 가격 표시 분기 규칙

**배경**: 정가+판매가가 모두 있는 경우 정가에 취소선 + 판매가 강조 세트가 임팩트가 크지만, 판매가만 있으면 단독 강조가 자연스러움. 단일 규칙으로 강제하면 정보 부족 시 어색해짐.

**확정**: 권위 필드의 가격 구성에 따라 분기.
- 정가 + 판매가 모두 있음 → `text_elements`에 `role: "price_original"`(끝에 "(취소선)" 표기) + `role: "price_sale"`
- 판매가만 → `role: "price_sale"` 또는 `role: "badge"` 단독

### 9-5. UI 복붙 블록 정정 — 섹션 이름 제외 + design_notes 포함

**배경**:
- 섹션 이름(`[섹션 1] 오프닝—결핍직격` 등)은 마케팅 전략 레이블. GPT한테 전달하면 자율성·창의성을 떨어뜨림.
- 반면 `design_notes`(부정 명령, 인물 외형, 아웃포커스 조건부, 가격 분기 등)는 **GPT한테 전달돼야 의미가 있음**. 그동안 누락되어 있었던 시스템적 버그.

**확정** ([pages/2_pipeline.py:1855](pages/2_pipeline.py:1855) `_build_section_copy_block`):
- 섹션 이름 헤더 제거 → 카드 UI 헤더에만 노출, GPT 복붙에서 제외
- `design_notes` 복붙 블록에 포함 추가
- `text_elements`에 `role` 표기 추가 (`[price_original]`, `[price_sale]`, `[headline]` 등 GPT가 시각 위계 판단에 활용)

### 9-6. design_notes 용도 정제 (C'안)

**배경**: design_notes에 메타 디렉터 지시("스킵존 카피 절대 금지", "저관여 제품 길게 끌지 말 것" 등)와 GPT용 정보(포토리얼, 부정 명령, 외형 통일)가 섞이는 경향. 메타 지시는 GPT한테는 노이즈.

**선택지 비교**:
- C안 (규칙 추가) — instruction.md에 "design_notes는 GPT용. 메타 지시 금지" 명문화. 단, LLM에 추가 판단 부담 발생.
- **C'안 (예시만 정리)** — core.md JSON 예시의 design_notes에서 메타 지시 두 줄만 제거. LLM이 예시를 따라 자연스럽게 GPT용 정보만 담게 됨. 추가 규칙·판단 부담 없음.

**확정**: **C'안 채택**. core.md JSON 예시 [core.md:167](agents/04_1_image_direction/core.md:167)에서 "스킵존 카피 절대 금지. 첫 컷이 결핍 장면이므로 과도한 밝음 지양." 제거.

### 9-7. size_px fallback 제거

**배경**: [2_pipeline.py:1864, 1932](pages/2_pipeline.py:1864)에 `canvas.get("width_px") or canvas.get("size_px")` 형태의 fallback 코드가 있었음. 어느 버전에서 사용했던 필드명인 듯하나 현재 출력 구조엔 없음.

**확인**: DB(`엠군_이미지디렉션` 테이블) 22개 디렉션 행에서 `size_px` 사용: **0건**, `width_px` 사용: 7건. 죽은 fallback.

**확정**: fallback 제거.

### 9-8. GPTs 운영 보류 (8장 결정 정정)

**배경**: 8장에서는 "GPTs 지침에 SKILL.md 등록"이 확정 워크플로우였으나, "지침에 규칙이 고정되면 GPT가 매번 강제 적용하려 해 창의성을 해친다"는 우려로 보류 결정.

**현재 운영 방식**:
- GPTs **미사용** (보류 중)
- SKILL_gpt_image.md는 **새 세션 시작 시 1회만** 첨부 (제품 사진 + 디자인 시스템 + 첫 섹션 브리프와 함께)
- 2번째 섹션부터는 섹션 브리프만 입력 (GPT가 세션 컨텍스트 유지)
- 한 세션 = 한 제품 원칙 유지

**SKILL.md의 첫 부분 안내**("GPTs 지침에 등록")는 사용자가 GPTs 운영 최종 결정 후 정리 예정.

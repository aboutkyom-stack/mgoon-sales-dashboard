# 자동화 판매 시스템 — 마스터 플랜

> 엠군 강의 기반 온라인 쇼핑몰 자동화 판매 시스템.
> 쿠팡 / 네이버 / 11번가 / 옥션 / 지마켓 / 도매꾹 등 멀티 플랫폼 운영.

---

## 1. 시스템 목적

- DB에 등록된 제품(R)에 대해 부분엠군들이 순차적으로 작업을 이어받아 판매 전략 전체를 자동 생성
- 타겟 하나 = 독립된 파이프라인 (타겟별 상세페이지·채널 전략 분리 생성)
- 그물을 넓게 치는 전략 → 타겟 후보 전부 발굴 후 우선순위 제시, 사용자가 선택

---

## 2. 운영 모드

| 모드 | 설명 | 시작 M군 |
|------|------|----------|
| **R 있음** | 판매할 아이템이 확정된 상태 | 결핍·타겟 M군 |
| **R 없음** | 아이템 탐색부터 시작 | (미구현 — 추후 추가) |

현재는 **R 있음 모드**만 구현.

---

## 3. 부분엠군 목록

| 번호 | 이름 | 담당 강의 그룹 | 역할 |
|------|------|--------------|------|
| 01 | deficit_target | G1 + G2 + G3 | 결핍 유형·깊이·타겟 캐릭터 발굴 |
| 02 | positioning | G5 + G6 | CV 설정, 가치더하기, 경쟁 포지셔닝 |
| 03 | naming | G9 일부 | 제품명/키워드 네이밍 (옵셔널) |
| 04 | detail_page | G4 | 설득프로세스 5가지 적용 상세페이지 |
| 05 | channel | G9 | 블로그·카페·SNS 등 물길 전략 |

- **종합 M군(core.md)**: 큰 흐름 파악·그룹 연계호출 시 참조용. 매번 로드하지 않음.
- **원본 페르소나**: `persona/mgoon/` — 절대 수정 금지. 부분엠군 제작 시 복사해서 사용.

---

## 4. 파이프라인 — R 있음 모드

```
[1] 제품(R) DB 등록
        ↓
[2] 결핍·타겟 M군 (01_deficit_target)
    → 타겟 후보 전체 발굴
    → 각 타겟별: 핵심 결핍 / 결핍 유형 / 깊이(높음·중간·낮음) / 추정 범위
    → DB 저장 (targets 테이블)
        ↓
[3] ★ 사용자 확인
    → 시스템이 타겟 목록 + 추천 우선순서 제시
    → 사용자가 실행할 타겟 선택 및 순서 결정
    → (안정화 후) 자동 승인 옵션 추가 예정
        ↓
[4] 선택된 타겟별로 독립 파이프라인 순차 실행:
    ┌──────────────────────────────────────────┐
    │ 타겟 1                                   │
    │  → 포지셔닝·가치 M군 (02_positioning)   │
    │  → 네이밍 M군 (03_naming) [옵셔널]      │
    │  → 상세페이지 M군 (04_detail_page)      │
    │  → 물길 M군 (05_channel)                │
    └──────────────────────────────────────────┘
    ┌──────────────────────────────────────────┐
    │ 타겟 2                                   │
    │  → ... (동일 구조)                       │
    └──────────────────────────────────────────┘
    ※ 각 단계 완료 시 DB 저장 후 다음 M군으로 이어받음
    ※ 현재 단계: 각 M군 완료 시 사용자 확인 후 진행
```

## 5. DB 구조 (개념)

```
products (제품 기본 정보)
  └── targets (타겟별 분석) [1:N]
        └── positioning (포지셔닝·가치) [1:1]
              └── detail_page (상세페이지) [1:1]
                    └── channels (채널별 물길 전략) [1:N]
```

- 현재: 로컬 SQLite
- 공용 DB 전환 시: Supabase(PostgreSQL) 유력 후보 → 아래 공용화 계획 참조

## 8. 공용화 계획 (메모)

> 현재는 로컬 개발 우선. 완성 후 동료와 함께 사용할 예정.

- **Streamlit Cloud + GitHub**: URL 하나로 동료가 동일 화면 접근. 배포 자동화.
- **Supabase**: PostgreSQL 기반 무료 티어. 어디서든 동일 DB 접근.
- **주의**: API 키·DB 연결 정보는 Streamlit Cloud Secrets에 등록. 코드에 절대 포함 금지.
- **대안 검토**: Google Sheets + `st.connection("gsheets")` — 데이터 양 적고 비개발자 동료라면 고려 가능.

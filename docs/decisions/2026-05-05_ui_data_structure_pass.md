## 1. 결정사항 요약 (코드/DB 영향 큰 것만)

### A. 모델 설정 — Claude/Gemini 분리 → 주/비교 통합

**Why**: Claude·Gemini 분리 셀렉트는 단계마다 "두 family를 다 쓰는가?"를 강제했다. 비교가 필요 없는 단계도 무조건 양쪽 모델 호출 → 비용·UX 낭비.

**무엇을**:
- 단계당 단일 셀렉트 2개 (주 사용 모델 + 비교 모델). 비교는 단계별 on/off 토글.
- 모델 후보 통합 리스트 (Claude+Gemini 한 자리), 순서: sonnet → opus → haiku → 2.5-flash → 2.5-pro. **gemini-2.0-flash 후보 제거** (2.5가 모든 면에서 우위, 사용 이유 없음).
- 단계 00·01·02 → **00~05 전 단계** 확장.

**자동으로 따라온 변경**: 비교 off 시 LLM이 None 모델로 호출되어 503 에러 → `pipeline/llm.py`에 None 스킵 추가 + UI 단일 컬럼 분기 (`_result_layout()` 헬퍼).

### B. 가격 영역 — 권위 필드 명확화

**Why**: 기존 `실제가`/`평균도매가`는 "실제로 받는 돈" / "원가"를 의미하지만 운영자가 헷갈림. 또한 "현재 판매 중인 가격"을 별도로 적을 곳이 없어 LLM이 박스 인쇄가나 평균값을 판매가로 오인 (2026-05-02 결정 로그 §0의 사실 비약 패턴).

**무엇을**:
- DB 컬럼 rename: `실제가` → `실제받는가격` (운영자가 받는 입금액), `평균도매가` → `평균입고가` (입고 단가).
- 신규 `온라인판매가격 INTEGER` (현재 판매가, BOOLEAN `온라인판매가능`과 헷갈리지 않도록 "가격" 명시).
- `agents/_shared/data_contract.md` 권위 우선순위에 `온라인판매가격` 최상위 명시. LLM이 가격 언급 시 이 필드 우선.

**검증**: `python db/run_rename_가격_컬럼.py` 1회 실행 완료. 운영 DB에 신컬럼 존재.

### E. 판매자 메모 / 기타 제품 특징 영역

**Why**: 기존 단일 `특징` 컬럼이 "비전패스 자동 추출 결과 + 판매자 수동 메모 + 외부 시스템에서 마이그레이션된 마케팅 카피"를 모두 담아 출처 구분 불가능. 또 "판매자만 보고 시스템은 무시할 메모"가 없어 판매자가 시스템 입력을 꺼리게 됨.

**무엇을** (4개 영역으로 분리):

| 위치 | 영역 | 출처 | DB 필드 | 시스템 참고(01-05) |
|---|---|---|---|---|
| 스펙 탭 | 🏷️ 기타 제품 특징 | 비전패스 자동 + 판매자 수정 | `제품특징_bullet` JSONB | ✅ |
| 스펙 탭 | 연관 키워드 | 비전패스 + 판매자 | `키워드` (재활용) | ✅ |
| 판매자 정보 탭 | 📌 제품 특징 추가 | 판매자 수동 | `제품특징_추가` TEXT | ✅ |
| 판매자 정보 탭 | 🗒️ 판매자 개인 메모 | 판매자 수동 | `판매자메모` (재활용) | ❌ |

`판매자메모`는 시스템 미참조 — D 단계의 `excluded_fields_*` 기본값에 포함되도록 설계.

기존 `특징` 컬럼은 이관 후 일정 기간 유지 (안전), 추후 별도 마이그레이션으로 제거.

**검증**: `python db/run_add_제품특징_영역.py` 1회 실행 완료. 기존 `특징` 데이터 3건이 줄바꿈 split → JSON 배열로 자동 이관됨.

### E+. 시각설명 → 스펙 + bullet 자동 추출 통합

**Why**: 사용자 피드백 "비전패스 한 번에 다 채워졌으면. 별도 추출 실행 버튼은 번거로움."

기존 흐름: 비전패스(시각설명 생성) → 시각설명 저장 → **스펙 탭에서 추출 실행 별도 클릭** → 미리보기 → **폼 적용 별도 클릭** → 저장 (총 4회 사용자 트리거).

**무엇을**:
- `pipeline/vision_merge.py:extract_스펙()`: 단일 LLM 호출에서 SPEC_FIELDS + `_특징_bullet` 함께 반환. 별도 `extract_특징_bullet` 함수 제거.
- `pages/2_product_edit.py`: 시각설명 저장 핸들러 3곳에 `_auto_apply_after_vision()` 헬퍼 호출 삽입. 시각설명 저장 직후 자동 LLM 호출 → "빈 필드만 채우기" 모드로 SPEC 자동 반영 + bullet 중복 제거 후 추가 → 즉시 DB update → 토스트 `"저장 완료! 자동 반영: 스펙 N개 / bullet M개"`.

**안전장치**:
- 빈 필드만 채우기 (손으로 채운 값 보존)
- bullet 중복 제거 (덮어쓰기 X)
- 전체 덮어쓰기는 기존 "🤖 추출 실행" expander의 라디오로 수동 진행 (라벨에 "수동 재실행" 명시)
- LLM 실패 시 토스트 경고만, 시각설명 저장 자체는 성공 유지

**Trade-off**: 시각설명 저장 시 LLM 비용 1회 자동 발생. 그러나 이전엔 사용자가 어차피 추출 실행 버튼을 따로 눌러야 했으므로 비용 증가 없음 — 트리거만 자동화됨.

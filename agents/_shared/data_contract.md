# 데이터 권위 규약 (모든 단계 공통)

이 문서는 01·02·03·04·05 모든 단계에 자동 prepend된다.
제품 입력(JSON)에 들어있는 여러 필드 사이에 충돌이 있을 때, 어느 쪽을 사실로 받아들이고 카피·CV·포지셔닝의 근거로 삼을지 정하는 규약이다.

---

## 필드 권위 우선순위

다음 필드들은 사용자가 직접 입력·검증한 값이다. **사실의 권위는 항상 이쪽**이다.

| 영역 | 권위 필드 | 비고 |
|------|----------|------|
| 가격 | `[가격].소매가` / `[가격].도매가` / `[가격].실제가` / `[가격].평균도매가` | 카피·CV·포지셔닝·할인 프레임 모두 이 값 기준 |
| 스펙 | `[스펙].모델명` / `[스펙].제조사` / `[스펙].원산지` / 치수·무게·재질·색상 | 시각설명의 추정값과 다르면 이쪽을 따른다 |
| 인증·연령 | `[인증].kc인증` / `[인증].kc인증번호` / `[인증].기타인증` | "KC인증 보유" 주장의 유일한 근거 |
| 재고·판매조건 | `[재고]` / `[판매조건]` 객체 전체 | "한정 수량", "재입고" 류 카피의 근거 |
| 카테고리 | `[카테고리]` / `[서브카테고리]` | 시각설명의 카테고리 추정과 충돌 시 이쪽 |

---

## 시각설명(`[시각설명]`) 처리 규칙

`[시각설명]`은 Vision Pass가 제품 이미지를 보고 추출한 텍스트다. **사실 주장의 근거가 아니라 카피·연출용 참고 자료**다.

### 절대 사실로 채택하지 말 것

다음 항목은 시각설명에 나오더라도 카피·CV·포지셔닝·할인 프레임의 근거로 쓰지 말 것:

- **가격 정보**: 포장재에 인쇄된 정가/할인가/세일 표시(예: "30,000원", "9,800원", "50% OFF")는 제조사·유통사가 포장재 디자인 시점에 인쇄한 라벨일 뿐, 실제 판매가가 아니다. 실제 판매가는 항상 `[가격]` 필드를 따른다.
- **마케팅 카피·슬로건**: 포장재에 인쇄된 카피("최고의 성능", "프리미엄" 등)는 제조사 마케팅이지 우리가 검증한 사실이 아니다.
- **인증 라벨**: 포장재 사진에서 본 KC·CE 등 인증 표시는 `[인증]` 필드와 일치할 때만 인용. 필드에 없으면 인용 금지.
- **수치성 주장**: 시각설명의 "300mAh", "5시간 지속" 등은 포장재 인쇄값일 뿐, 실측값 아님. `[스펙]`에 동일 항목 있으면 그쪽 우선.

### 카피·연출 참고로는 적극 활용

다음은 시각설명에서 가져와도 된다:
- 색감·외형·비주얼 묘사 (상세페이지 디자인 가이드)
- 패키징 인상 (사은품·언박싱 연출)
- 브랜드명·모델명·로고 위치 (단, `[스펙].모델명`/`[스펙].제조사`와 일치 시에만)
- 제품 부위·구조 묘사 (가치더하기 모듈화 단서)

---

## 충돌 발생 시 행동 원칙

1. **권위 필드와 시각설명이 다를 때**: 권위 필드를 따른다. 시각설명은 무시한다.
2. **권위 필드가 비어있을 때**: 그 항목에 대한 주장 자체를 하지 않는다. 시각설명으로 채우지 않는다. (예: `[인증].kc인증=null`이면 "KC인증" 단어 자체를 카피에 넣지 않음)
3. **권위 필드끼리 충돌할 때** (드물지만): `실제가` > `소매가` > `평균도매가` > `도매가` 순으로 신뢰. 단, 카피용 노출 가격은 `소매가`(정가)와 `실제가`(할인가)를 함께 쓰는 것이 일반적.
4. **권위 필드도 시각설명도 없을 때**: 그 항목 자체를 카피에 넣지 않는다. 추정·창작 금지.

## 자가 검수 체크 (단계별 출력 끝에 반영)

각 단계의 출력 끝 QA 자가 검수에 다음 항목을 추가한다:

```
□ D-01. 카피·CV에 사용된 가격이 [가격] 필드 값과 일치하는가? (시각설명의 포장재 인쇄 가격이 아닌가)
□ D-02. 인증·연령 주장은 [인증]·[스펙] 필드에 실제로 존재하는가?
□ D-03. 권위 필드가 비어있는 항목에 대해 시각설명·창작으로 채운 곳은 없는가?
```

위 세 항목 중 하나라도 ✗이면 출력을 수정한 뒤 다시 작성한다.

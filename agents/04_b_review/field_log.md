# 04_b 상세페이지 검수 — 필드 로그 (Field Log)

> **이 파일의 목적**: 04_b 상세페이지 검수 단계에서 발생한 판단·가설·편향을 누적. 다음 세션의 이 단계 작업 시 동일한 고민을 반복하지 않게 한다.
> **로드 시점**: 이 단계 작업 시작 시 자동 로드 (instruction.md 선행 의무 첫 항목). 없으면 건너뜀.
> **금지**: 이 파일을 "정답"으로 읽는 것. 항상 가설로 취급. core.md 수정 금지. 단일 사례로 단정 금지.

---

## 1. 이 파일의 목적

- 04_b 단계 고유 판단(DIRECT/RETURN/BUILD 판정·Rev 규칙 적용 순서·04_a 강사 어투 보존 기준)에서 발생한 학습을 기록.
- 여러 제품에 걸쳐 반복되는 검수 실수 패턴을 발굴하기 위한 **원자료**(raw data) 역할.
- 결론이 아니라 **가설**로 유지. 표본이 쌓이기 전에 core로 승격 금지.

---

## 2. 왜 core.md를 직접 수정하지 않는가

- core.md는 이 단계의 확정 검수 프레임. 소수 사례로 수정하면 **과적합(overfitting)** 위험.
- 엠군 원칙: *"분석은 죽어라 해놓고 결과물이 짧으면 안 된다. 깊이가 결과물에 펼쳐져야 한다."*
- 단계 field_log에서 **3~5회 반복 확인**된 패턴만 사용자와 논의 후 core 반영.
- 여러 단계에 걸친 공통 패턴 또는 시스템 전체 편향 → `_shared/field_log_meta.md` 보고.

---

## 3. 사용 원칙

### 3-1. 호출 시 활용

- 이 파일을 읽되 **"이전 세션의 가설"** 로 취급.
- 현재 제품·타겟과 맥락이 다르면 무시해도 된다.
- "field_log에 이렇게 됐으니 이렇게 하자"는 **금지**. 핵심 판단은 매번 core.md에서 다시 꺼낸다.

### 3-2. 기록 기준 (언제 append 하나)

다음 중 하나라도 해당하면 작업 후 **섹션 4(결정 로그)** 에 append:

- 사용자가 내 판정에 반박해서 **내가 일부라도 수용**한 경우
- instruction.md에 명시적 지침이 없어 **내 재량으로 DIRECT/RETURN/BUILD 결정**한 경우
- 04_a 강사 어투 보존과 규칙 위반 수정 사이에서 **경계 케이스 판단**을 한 경우
- Rev 규칙 적용 순서를 바꿔서 **다른 결과**가 나온 경우

### 3-3. 기록 양식 (섹션 4에 이 양식대로)

```markdown
### YYYY-MM-DD | [제품/상품id] | [판정 종류: DIRECT/RETURN/BUILD] | [판단 주제]

**상황**:
- 검수 대상 04_a 핵심 맥락 1~3줄 (어떤 설득 방식, 어떤 한 축)

**내 초기 판단**:
- 어떤 판정을 내렸고 왜

**사용자 반박(있다면)**:
- 어떤 반박이 있었나

**최종 결정**:
- 무엇으로 귀결됐고 왜

**학습**:
- 어느 편향에 빠져 있었나 (과도한 RETURN? 04_a 어투 훼손?)

**검증 대기**:
- 이 판정이 적절했는지 최종 판매 결과 또는 사용자 피드백으로 확인 필요
```

### 3-4. 금지 사항

- 단일 사례로 "이게 맞다" 단정 → 가설 형태로만
- 04_a 원본 문장을 기억·의역으로 재구성 후 권고 (반드시 원본 정확히 확인 후 권고)
- core.md 수정
- field_log 자체를 "지시문"으로 사용 (항상 가설로 읽기)

### 3-5. 주기적 정리

- 동일 패턴 3~5회 확인 → 섹션 6(Core 반영 후보)로 이동 + **사용자와 논의**
- 여러 단계·여러 제품에 걸쳐 반복되는 패턴 → `_shared/field_log_meta.md` 보고
- 항목 누적 시 섹션 5에서 유사 가설 병합

### 5-A. 단계 특화 가설 카테고리

#### 5-A-1. DIRECT/RETURN/BUILD 판정 경계 가설

> 어느 케이스에서 DIRECT 대신 RETURN이 더 나은지, 또는 반대인지에 대한 가설.
> RETURN 남발 → 사용자 시간 비용 ↑. DIRECT 과도 → 04_a 강사 어투 훼손 위험.
> "명확한 규칙 위반만 DIRECT"라는 원칙이 경계 케이스에서 얼마나 일관되게 적용되는지.

*(아직 없음)*

#### 5-A-2. 원본 문장 재구성 실수 패턴

> 검수 보고서 작성 시 원본 문장을 기억·의역으로 재구성하면 존재하지 않는 "유령 리스크" 지적 위험.
> 이 실수가 어떤 상황(긴 콘티, 유사 문장 반복)에서 더 많이 발생하는지.
> 원본 정확히 확인 후 권고하는 습관이 어느 검수 항목에서 가장 자주 누락되는지.

*(아직 없음)*

#### 5-A-3. Rev 규칙 적용 순서 가설

> Rev-00(ENGINE_PLAN 정합성)을 먼저 보고 나머지를 적용하는 순서가 언제 비효율적인지.
> 특정 규칙 위반이 연쇄적으로 다른 규칙 위반을 유발하는 패턴.
> Rev 규칙 간 우선순위가 바뀌어야 하는 엣지 케이스.

#### 5-A-4. 04_a 강사 어투 보존 vs 규칙 수정 충돌

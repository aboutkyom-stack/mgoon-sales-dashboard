# 이미지 디렉션 M군 검수 체크리스트 (04_1_image_direction)

> **로드 시점**: 디렉션 결과물이 나온 직후 검수 단계에서만.
> 작업 시작 시 로드하지 않는다 (토큰 낭비).
> **성격**: 판단 여지 없는 확정 규칙만. 재발 가능성 있는 실수 패턴 기록.

---

## 확정 규칙

### R-01. JSON 블록 부착 및 마커 정확성
본문 끝에 `---SECTIONS_JSON---` ... `---END_SECTIONS_JSON---` 블록이 정확한 마커로 부착됐는지 확인.

- **위반 예**: 마커 누락 / 오타 (`SECTION_JSON`, `IMAGE_JSON`) / 블록 안 JSON 문법 오류
- **통과 예**: 마커 정확 + JSON.parse 가능
- **근거**: 시스템이 이 블록만 파싱한다. 어긋나면 DB 저장·UI 표시 실패.

### R-02. 카피 확정성
모든 `text_elements[].content`가 조사·문장부호까지 확정된 최종 문구여야 한다.

- **위반 예**: "(예시) 명절에 손주 왔는데..." / "이런 느낌의 카피" / 괄호로 옵션 제시
- **통과 예**: "명절에 손주 왔는데, 할머니는 TV 보고 손주는 유튜브 보고 있진 않으셨나요?"
- **근거**: 디렉션은 확정안이지 콘티가 아니다. 디자이너·이미지 AI가 그대로 쓸 수 있어야 한다.

### R-03. 권위 필드 변형 금지
가격·인증번호·모델명은 [제품 정보]의 권위 필드를 한 글자도 바꾸지 말고 그대로 써야 한다.

- **위반 예**: "8000원" (콤마 누락) / "CB064R-5567002" (포맷 변형) / 모델명 임의 축약
- **통과 예**: "8,000원" / "CB064R556-7002" — 입력값 그대로
- **근거**: 인증·가격 표기 변형은 법적/마케팅 리스크.

### R-04. 섹션 수 — 일치 또는 분할 허용
04 콘티 표의 모든 행이 `sections` 배열에 빠짐없이 반영되어야 한다. 단, 한 행이 카피·요소 과부하로 한 컷에 안 담기면 2개 섹션으로 분할 가능하다(분할 시 `name`에 `-A`, `-B` 접미사로 출처 표시).

- **위반 예**: 콘티 7행 → 디렉션 5섹션 (2행 누락) / 디렉션 9섹션 (없는 섹션 창작 — 분할이 아닌 신규 기획)
- **통과 예**: 콘티 7행 → 디렉션 7섹션 (이름·순서 보존) / 콘티 7행 → 디렉션 8섹션 (5번째 행을 "기능설명-A", "기능설명-B"로 분할)
- **분할 기준**: `text_elements` 4개 초과 OR `composition`에 독립적으로 묘사할 영역이 2개 이상
- **근거**: 디렉션은 콘티의 변환이지 재기획이 아니다. 단, 한 컷에 다 담으려다 비율·여백이 깨지면 시각 품질이 무너지므로 동일 행을 2컷으로 풀 수 있는 여지를 둔다.

### R-05. 한국어 필드 충실성 + 시각 실행 위임
각 섹션의 `composition`, `background`, `mood`, `design_notes`가 한국어로 충실히 작성돼야 한다. 별도 `gpt_prompt` 필드는 만들지 않는다.
`text_elements`는 **`content`, `role` 2항목만** 사용한다. `position`, `size`, `color`, `weight`, `padding`, `background` 등 시각 실행 스펙은 일체 포함 금지.
`composition`은 "상단 1/3, 중단 1/3, 하단 1/3" 같은 등분 강제 표현 금지 — "상단에 제품, 하단에 Q&A·구매 영역" 식으로 영역만 자연어로 지시.

- **위반 예**: `text_elements`에 `position`/`size`/`color`/`weight`/`padding` 포함 / 별도 `gpt_prompt` 영문 필드 추가 / `composition`에 "상단 1/3 ... 중단 1/3 ..." 등분 강제 / 한국어 필드가 비거나 키워드만
- **통과 예**: `text_elements`에 `content`·`role` 2항목만 / `composition`이 영역 흐름을 자연어로 / 한국어 필드 충실
- **근거**: 좌표·픽셀·등분을 강제하면 GPT Image가 요소를 기계적으로 배치해 박스 테두리·글자 테두리·구획 분할이 투박해진다. 카피와 영역만 주고 시각 실행은 GPT Image의 디자인 감각에 위임해야 결과물 품질이 높다.

### R-06. 인물 일관성 명시
같은 인물이 여러 섹션에 등장하면 외형 묘사가 모든 섹션에서 동일해야 한다 (헤어, 옷 색, 나이대).

- **위반 예**: 섹션 1에선 흰머리 할머니, 섹션 4에선 검은머리 할머니 (같은 인물인데)
- **통과 예**: 모든 등장 섹션에서 "60대 한국인 할머니, 짧은 흰머리, 베이지 카디건" 통일
- **근거**: 일관성 깨지면 별도 인물처럼 보여 신뢰 무너진다.

### R-07. 부정 명령 명시
인물 등장 섹션의 `design_notes`에 `로고 없음. 워터마크 없음. AI 인공감 인물 없음.`이 명시돼야 한다.

- **위반 예**: design_notes에 부정 명령 누락 → AI 인공감 얼굴 / 임의 브랜드 로고 생성
- **통과 예**: `"design_notes": "... 로고 없음. 워터마크 없음. AI 인공감 인물 없음."`

### R-08. 후기 카피 인계
경험계(후기) 섹션의 카피는 04 콘티의 후기 카피를 그대로 가져와야 한다 (새 창작 금지).

- **위반 예**: 콘티의 후기 "할머니·손주 함께 웃은 순간 묘사 후기" → 디렉션에서 다른 후기로 창작
- **통과 예**: 콘티 카피 그대로 (조사·맞춤법만 다듬기 가능)

### R-09. 디자인 시스템 1회 선언
`design_system` 객체는 정확히 1번 정의되고, 각 섹션 마크다운 본문은 그 규칙(색·폰트·인물 외형)을 반복 적용해야 한다.

- **위반 예**: 섹션마다 색·폰트가 제각각 / design_system 누락
- **통과 예**: 전역 1회 선언 + 각 섹션 프롬프트가 그 색·폰트를 다시 명시

### R-10. 캔버스 크기·비율 명시 (너비 860 고정)
모든 섹션이 `canvas` 객체에 `width_px`, `height_px`, `ratio`를 모두 명시. 너비는 **860px 고정**, 비율은 4:5 / 2:3 / 1:1 세트 중 섹션 성격에 맞춰 선택.

- **위반 예**: canvas 누락 / "적당한 크기" 같은 모호한 표현 / `width_px`가 860이 아닌 값(1024, 1000 등) / 세트 외 비율(16:9, 3:4 등)
- **통과 예**: `{"width_px": 860, "height_px": 1075, "ratio": "4:5"}` / `{"width_px": 860, "height_px": 1290, "ratio": "2:3"}` / `{"width_px": 860, "height_px": 860, "ratio": "1:1"}`
- **세로 강조 카피 정합**: "○○보다 큽니다", "1.3미터입니다" 등 세로 길이 강조 카피가 있는 섹션은 반드시 2:3(860×1290) 선택. 4:5에 압축하면 시각이 카피와 따로 논다.

### R-11. 수치 비교 카피의 시각 비율 정합성
04 콘티에서 넘어온 카피에 수치 비교 표현("아이 키의 절반", "두 배", "어른 키만 합니다" 등)이 있으면, 그 비율이 이미지로 시각화될 때 실제 수치와 맞는지 확인. 틀렸으면 디렉션 카피를 수정해서 올바른 비율로 연출.

- **위반 예**: 제품 1.3m + 카피 "아이 키의 절반이 넘습니다" → 이미지에서 크레인이 아이 허리 높이로 표현됨. 실제로 1.3m는 아이(120cm)보다 크므로 비율 오류.
- **통과 예**: "아이 키를 넘습니다" / "아이보다 더 큰 크레인" 으로 카피 수정 후 이미지 연출
- **근거**: GPT Image는 카피의 수치 비율을 리터럴하게 시각화한다. 04에서 넘어온 카피가 틀렸더라도 04-1에서 바로잡지 않으면 그대로 이상한 비율의 이미지가 생성된다.

## 항목 추가 기준

다음 **모두** 만족할 때만 여기 추가:
- 동일/유사 실수가 **2회 이상** 발생
- **다른 제품·다른 세션에서 재발 가능**한 표면 실수
- **판단 여지 없이 규칙으로 확정 가능**

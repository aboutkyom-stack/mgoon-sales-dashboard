# 03 네이밍 — 필드 로그 (Field Log)

> **이 파일의 목적**: 03 네이밍 단계 작업에서 발생한 판단·가설·편향을 누적. 다음 세션의 이 단계 작업 시 동일한 고민을 반복하지 않게 한다.
> **로드 시점**: 이 단계 작업 시작 시 자동 로드 (instruction.md 선행 의무 첫 항목). 없으면 건너뜀.
> **금지**: 이 파일을 "정답"으로 읽는 것. 항상 가설로 취급. core.md 수정 금지. 단일 사례로 단정 금지.

---

## 1. 이 파일의 목적

- 03 단계 고유 판단(분류별 네이밍 전략·타겟 언어 선택·상표 빈틈 발굴·시리즈 확장성 검토)에서 발생한 학습을 기록.
- 여러 제품에 걸쳐 반복되는 패턴을 발굴하기 위한 **원자료**(raw data) 역할.
- 결론이 아니라 **가설**로 유지. 표본이 쌓이기 전에 core로 승격 금지.

---

## 2. 왜 core.md를 직접 수정하지 않는가

- core.md는 이 단계의 확정 사고 프레임. 소수 사례로 수정하면 **과적합(overfitting)** 위험.
- 엠군 원칙: *"네이밍만 잘 해도 반은 먹고 들어갑니다."*
- 단계 field_log에서 **3~5회 반복 확인**된 패턴만 사용자와 논의 후 core 반영.
- 여러 단계에 걸친 공통 패턴 또는 시스템 전체 편향 → `_shared/field_log_meta.md` 보고.

---

## 3. 사용 원칙

### 3-1. 호출 시 활용

- 이 파일을 읽되 **"이전 세션의 가설"** 로 취급.
- 현재 제품·타겟과 맥락이 다르면 무시해도 된다.
- "field_log에 이렇게 됐으니 이렇게 하자"는 **금지**. 핵심 판단은 매번 core.md에서 다시 꺼낸다.

### 3-2. 기록 기준 (언제 append 하나)

다음 중 하나라도 해당하면 작업 후 **섹션 4(결정 로그)** 에 append:

- 사용자가 내 분석에 반박해서 **내가 일부라도 수용**한 경우
- instruction.md에 명시적 지침이 없어 **내 재량으로 결정**한 경우
- 타겟 언어 vs 조어·상표 빈틈 공략 여부에서 **실험적 접근**을 한 경우
- 분류별(제품명/브랜드명/블로그제목/상품명) 전략에서 **재량으로 결정**한 경우

### 3-3. 기록 양식 (섹션 4에 이 양식대로)

```markdown
### YYYY-MM-DD | [제품/상품id] | [분류] | [판단 주제]

**상황**:
- 분류·CV·타겟 지능 등 핵심 맥락 1~3줄

**내 초기 판단**:
- 무엇을 선택했고 왜

**사용자 반박(있다면)**:
- 어떤 반박이 있었나

**최종 결정**:
- 무엇으로 귀결됐고 왜

**학습**:
- 어느 편향에 빠져 있었나

**검증 대기**:
- 04 카피 일관성 또는 05 키워드 연계 결과로 무엇을 확인해야 하나
```

### 3-4. 금지 사항

- 단일 사례로 "이게 맞다" 단정 → 가설 형태로만
- KIPRIS 미확인 상태에서 "이 이름 안전함" 단정
- core.md 수정
- field_log 자체를 "지시문"으로 사용 (항상 가설로 읽기)

### 3-5. 주기적 정리

- 동일 패턴 3~5회 확인 → 섹션 6(Core 반영 후보)로 이동 + **사용자와 논의**
- 여러 단계·여러 제품에 걸쳐 반복되는 패턴 → `_shared/field_log_meta.md` 보고
- 항목 누적 시 섹션 5에서 유사 가설 병합

### 5-A. 단계 특화 가설 카테고리

#### 5-A-1. 타겟 언어 vs 조어(造語) 선택 가설

> 타겟이 실제로 쓰는 언어(검색어)를 그대로 쓸 때 vs. 내가 만든 전문 용어(조어)를 쓸 때의 효과 차이.
> 타겟 언어는 SEO 유리, 조어는 포지셔닝 독점 가능성. 어느 제품에서 조어가 먹히는지.
> 욕구에 맞닿지 않는 조어는 납득 불가 → 역효과.

*(아직 없음)*

#### 5-A-2. KIPRIS 결과 반영 패턴

> KIPRIS 검색 후 후보를 갱신하는 과정에서 어떤 수준의 유사성이 차단 사유가 되는지.
> 동일 음가·유사 의미·동일 카테고리 등 기준이 케이스마다 다를 수 있음.
> 사용자가 KIPRIS 결과를 어느 정도까지 허용하는지 파악.

*(아직 없음)*

#### 5-A-3. 분류별 난이도 가설

> 제품명·브랜드명·블로그제목·상품명 중 어느 분류에서 수정 요청이 많은지.
> CV가 명확할수록 네이밍이 쉬워지는 원칙이 분류별로 다르게 작동할 수 있음.
> 블로그제목은 05 키워드가 없으면 임시 작성 → 재작업 빈도가 높은 편.

*(아직 없음)*

#### 5-A-4. 05 키워드 연계 일관성 가설

> 03에서 정한 블로그제목·상품명이 05 채널 작업 후 키워드와 얼마나 일치하는지.
> 05 이후 재호출 빈도와 그 때 변경 폭 추적.
> 02 CV가 정교할수록 03→05 연계 일관성이 높아지는지.

## 작업 절차

### 1) 선행 정독

- `agents/02_positioning/field_log.md` — 이전 세션 가설 (없으면 건너뜀)
- `agents/_shared/data_contract.md` — 권위 필드 규약
- `agents/_shared/강의원본_매핑.md` § 02 — 정독 대상 강의 인덱스

### 2) 핵심 강의 ⭐ (의무 정독, `_3summary` 전부)

1. `1주차 15 16 핵심가치 일이관지`

3. `1주차 19 20 포지셔닝 삼성은 우리의 가족`
4. `1주차 21 22 핵심가치 다 걷어내고 남는게 CV`
5. `1주차 23 24 25 소비자 구매편익`
6. `1주차미피 08 09 CV 틀리면 인생 망함 사조참치 미션`
7. `2주차 13 가치더하기 300원 귤`

9. `3주차미피 11 이동시키면 경쟁 없는 시장`

### 3) 보강 정독 (제품 특성에 맞는 것)

- 가격 포지셔닝 필요 → `1주차미피 17·18 RCS2 비싸게 파는 방법 1·2`
- 시장 리더 대응 → `2주차 02 시장리더 쫄 필요 없는 이유`
- 자연 해결 제품 → `2주차미피 06 호박즙·멍크림`
- 단점 장점화 → `3주차미피 10 단점을 장점으로`

### 4) 입력 자료 로드

- `items/item_<id>/01_deficit_target.md` — 선택된 타겟 + TARGETS_JSON
- `items/item_<id>/00_vision_pass.md` — VP 결과 (가치더하기·같은 군 차별화 단서)
- 상품 테이블 권위 필드 (가격·인증·스펙·판매자특성)

### 5) 분석 → POSITIONING_JSON 생성

아래 [자동화 모드 출력 형식 명세 ⭐] 그대로. 마크다운 7항목 + 강사 멘트 + POSITIONING_JSON 블록.

**JSON 필드 결정 가이드는 아래 출력 형식 명세의 본문↔JSON 매핑 표 참조.**

### 6) 자가 검수 + 다음 단계 안내

```
"02 포지셔닝 완료. 검토 후 진행할 부분 있으면 말씀.
다음 단계는 03 네이밍입니다.

'엠군: 네이밍 / [상품id] / [분류]'로 호출하세요. 분류는 제품명/브랜드명/블로그제목/상품명 중 선택."
```

**단계 흐름 (참고)**:
```
02 포지셔닝 → 03 네이밍 → 04_a 상페 작성 → 04_b 검수 → 04_1 이미지 → 05 채널
```

## ⭐ 트리거 규칙

### T-01. 구매편익 분류 (필수)
타겟의 구매편익이 기능재/경험재/상징재 중 어디인지 결정 시:
→ `1주차 23 24 25 소비자 구매편익_2clean` 정독
→ 보강: `1주차미피 04 05 당궁모 구매편익 틀렸습니다_2clean`

### T-02. 상징제 vs 기능제 판정
경쟁 제품이 상징제이고 우리가 기능제 진입을 고려할 때:
→ `1주차미피 08 09 CV 틀리면 인생 망함 사조참치 미션_2clean` (반드시)
→ 사조참치 안심따개 실패 사례 흡수해 무모한 기능제 진입 경고

### T-03. 가치더하기 본격 활용
가치더하기 2~3개 도출 시:
→ `2주차 13 가치더하기 300원 귤_2clean`
→ `3주차 06 정육각 vs M군 치킨집_2clean`
→ `3주차 07 패딩 왜 비싼지_2clean` (다층 가치)

### T-04. 핵심가치 이동 전략
`rule_engine_flags.구매핵심가치_이동_필요 = true`로 결정 시:
→ `3주차미피 11 이동시키면 경쟁 없는 시장_2clean`
→ 1·2순위 CV가 정말 포화·평준화됐는지 사례 검증

### T-05. 같은 군 차별화 가치더하기
같은 군 안 사실 기반 우월성 도출 시:
→ `3주차 06 정육각 vs M군 치킨집_2clean` 정독
→ 비전패스 사실 레이어 재훑기

### T-06. 자연 해결 제품
제품 본질 = "자연적 해결"로 분류 시:
→ `2주차미피 06 자연적으로 좋아지는 호박즙·멍크림_2clean`
→ "효과 주장 X, 심리 안정·속도 가속" 카피 방향 확보

### T-07. 가격 포지셔닝 (싸게/비싸게)
가격 전략 명시 시:
→ 싸게 → `1주차미피 17 RCS2 비싸게 파는 방법 1_2clean` (싸게 가는 논리도 여기)
→ 비싸게 → `1주차미피 18 RCS2 비싸게 파는 방법 2_2clean`

### T-08. 시장 리더 대응
경쟁 강도 = 레드 + 시장 리더 존재 시:
→ `2주차 02 시장리더 쫄 필요 없는 이유_2clean`

### T-09. 단점 장점화 활용
가치더하기에 "단점의 장점화" 포함 시:
→ `3주차미피 10 단점을 장점으로_2clean`

### T-10. category_objections 발굴 (필수) ⭐

POSITIONING_JSON의 `category_objections` 필드는 04_a 핵심 반론 선제 처리 블록(풀세트 3.5단계)의 직접 입력. 비워두면 04_a가 블록 못 만들거나 부정확하게 만듦.

절차:
1. **사용자 입력 우선** — 호출 시 사용자가 "이 카테고리에서 소비자가 가장 의심할 점"을 알려줬는지 확인
2. **AI 추론 (사용자가 모르면)** — 카테고리 도메인 지식 + 01 결핍 + 00 VP를 기반으로 자가 자문: 이 카테고리 처음 본 소비자가 가장 먼저 떠올리는 의심·반론은? (예: 효과 정말 있나? 학습되지 않나? 안전한가? 다른 게 더 낫지 않나?)
3. **추론도 불가** — `source: "모름"`으로 명시 + 사용자에게 추가 입력 요청
4. **강의 보강** — 카테고리 의심이 "단점의 강점화"로 해소 가능하면 `3주차미피 10_2clean` 정독
5. **강의 보강** — 카테고리 의심이 "결제 불안" 류면 `2주차미피 09 고객 안심 결제_2clean` 정독

→ 도출 개수 1~3개. 가장 결정적인 1개를 1순위.
→ 비둘기 퇴치제 예: "익숙해지면 효과 없어지지 않나?" — 학습력이라는 도메인 지식이 필요. AI 추론 가능.
→ 침구 청소기 예: "이거 살균 진짜 되나?" — 카테고리 공통 의심. AI 추론 가능.

## 작업 원칙

- 구매 CV는 **소비자 시각**으로 (판매자 시각 X)
- 포지셔닝 그래프 X·Y축은 **선호도 언어** ("거리 접근성↑" O, "거리" X)
- 가치더하기는 욕구에 닿는 것만. **상한 3개**
- 같은 군 차별화는 사실 기반 우월성 있을 때만
- 구매핵심가치 이동 결정 시 근거 한 줄 명시
- 04 단계가 직접 사용할 `rule_engine_inputs` + `rule_engine_flags`를 정확히 채울 것
- 출력 끝에 자가 검수(R-01~R-09) + POSITIONING_JSON 블록 (아래 명세)

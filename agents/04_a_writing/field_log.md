# 04_a 상세페이지 콘티 — 필드 로그 (Field Log)

> **이 파일의 목적**: 04_a 상세페이지 콘티 작성 단계에서 발생한 판단·가설·편향을 누적. 다음 세션의 이 단계 작업 시 동일한 고민을 반복하지 않게 한다.
> **로드 시점**: 이 단계 작업 시작 시 자동 로드 (instruction.md 선행 의무 첫 항목). 없으면 건너뜀.
> **금지**: 이 파일을 "정답"으로 읽는 것. 항상 가설로 취급. core.md 수정 금지. 단일 사례로 단정 금지.

---

## 1. 이 파일의 목적

- 04_a 단계 고유 판단(ENGINE_PLAN 한 축 사슬 설계·헤드카피 선택·설득 방식 선택·내부 용어 노출 방지)에서 발생한 학습을 기록.
- 여러 제품에 걸쳐 반복되는 패턴을 발굴하기 위한 **원자료**(raw data) 역할.
- 결론이 아니라 **가설**로 유지. 표본이 쌓이기 전에 core로 승격 금지.

---

## 2. 왜 core.md를 직접 수정하지 않는가

- core.md는 이 단계의 확정 사고 프레임. 소수 사례로 수정하면 **과적합(overfitting)** 위험.
- 엠군 원칙: *"상세페이지 초반 1/3에 90%가 다 있어요. 첫 카피에서 결핍을 못 건드리면 끝난 거예요."*
- 단계 field_log에서 **3~5회 반복 확인**된 패턴만 사용자와 논의 후 core 반영.
- 여러 단계에 걸친 공통 패턴 또는 시스템 전체 편향 → `_shared/field_log_meta.md` 보고.

---

## 3. 사용 원칙

### 3-1. 호출 시 활용

- 이 파일을 읽되 **"이전 세션의 가설"** 로 취급.
- 현재 제품·타겟과 맥락이 다르면 무시해도 된다.
- "field_log에 이렇게 됐으니 이렇게 하자"는 **금지**. 핵심 판단은 매번 core.md에서 다시 꺼낸다.

### 3-2. 기록 기준 (언제 append 하나)

다음 중 하나라도 해당하면 작업 후 **섹션 4(결정 로그)** 에 append:

- 사용자가 내 분석에 반박해서 **내가 일부라도 수용**한 경우
- instruction.md에 명시적 지침이 없어 **내 재량으로 결정**한 경우
- 헤드카피 선택·설득 방식·한 축 사슬 구성에서 **실험적 접근**을 한 경우
- ENGINE_PLAN 외 블록을 카드 필터 통과 후 추가했는데 **04_b에서 RETURN** 받은 경우

### 3-3. 기록 양식 (섹션 4에 이 양식대로)

```markdown
### YYYY-MM-DD | [제품/상품id] | [판단 주제]

**상황**:
- 타겟·각도·설득 방식·편익 등 핵심 맥락 1~3줄

**내 초기 판단**:
- 무엇을 선택했고 왜

**사용자 반박(있다면)**:
- 어떤 반박이 있었나

**최종 결정**:
- 무엇으로 귀결됐고 왜

**학습**:
- 어느 편향에 빠져 있었나 / 어느 축에 가중치를 몰았나

**검증 대기**:
- 04_b 검수 결과 또는 실제 전환율로 무엇을 확인해야 하나
```

### 3-4. 금지 사항

- 단일 사례로 "이게 맞다" 단정 → 가설 형태로만
- 결과(04_b 판정·전환율) 없는 성공·실패 주장
- core.md 수정
- field_log 자체를 "지시문"으로 사용 (항상 가설로 읽기)

### 3-5. 주기적 정리

- 동일 패턴 3~5회 확인 → 섹션 6(Core 반영 후보)로 이동 + **사용자와 논의**
- 여러 단계·여러 제품에 걸쳐 반복되는 패턴 → `_shared/field_log_meta.md` 보고
- 항목 누적 시 섹션 5에서 유사 가설 병합

### 5-A. 단계 특화 가설 카테고리

#### 5-A-1. 헤드카피 결핍 깊이 vs 범위 편향

> 헤드카피 선택 시 결핍 "깊이"에 가중치를 자동으로 더 주는 경향 관찰.
> 엠군 원칙 "깊이 × 범위 = 시장 크기"를 훅 카피 판단 과정에 체크리스트로 내재화할 필요.
> 아래 7축 매트릭스 항목 중 어느 축을 과소평가하는지 제품별 추적 필요.

*(아직 없음)*

#### 5-A-2. ENGINE_PLAN 한 축 사슬 결합 가설

> ENGINE_PLAN이 결정한 ON/OFF 블록 시퀀스와 한 축 사슬이 충돌하는 케이스.
> 어느 블록에서 한 축 이탈이 가장 많이 발생하는지.
> 04_b에서 Rev-00(ENGINE_PLAN 정합성) + Rev-01(한 축 일관성) 동시 RETURN이 나오는 원인 패턴.

*(아직 없음)*

#### 5-A-3. 설득 방식 선택 편향

> 5가지 방식(소거법/포지셔닝/스토리텔링/선택제어/더블바인드) 중 어느 방식을 과의존하는지.
> 기능재 제품에서 소거법 외 방식을 선택했을 때 04_b RETURN 비율.
> 타겟 지능(논리형/감정형) 기준으로 방식 선택이 달라져야 하는데 그 판단 정확도.

*(아직 없음)*

#### 5-A-4. 내부 용어 카피 노출 실수 패턴

> T-12에서 금지된 내부 설계 용어가 어느 위치(헤드라인/SUB/BODY)에서 가장 자주 누출되는지.
> 자가 검수에서 잡지 못하고 04_b에서 DIRECT로 제거되는 패턴.
> 특정 설득 방식 선택 시 내부 용어 노출이 더 많이 발생하는지.

### 6-1. 발생 중인 프레임워크 (실험적)

**헤드카피 평가 7축 매트릭스** — 이 단계 작업 중 도출 가능한 실험적 평가 프레임.

| # | 축 | 설명 |
|---|---|---|
| 1 | 통찰 2단계 적중도 | 타겟이 "내 얘기" 될 확률 |
| 2 | 결핍 강도 | 본전심리 발동 깊이 |
| 3 | 수습 가능성 | 다음 블록에서 채울 수 있는 깊이인가 |
| 4 | 다음 블록 연결성 | 결핍 심화로 이어지는가 |
| 5 | 구체성·시각화 | 머리에 그림 그려지는가 |
| 6 | 이탈 리스크 | 특정 조건 타겟 이탈 가능성 |
| 7 | 다른 블록 중복 | 뒤쪽 블록 기능과 겹치나 |

- **주의**: 7축 중 어느 축에 가중치를 얼마나 줄지는 **매번 재판단**. 고정 공식 아님.
- **현재 상태**: 실험적 프레임. 3~5개 제품 적용 후 패턴 확인 필요.
- 반영 후보 조건: 이 매트릭스를 명시 적용했을 때 04_b RETURN 감소 확인 후.

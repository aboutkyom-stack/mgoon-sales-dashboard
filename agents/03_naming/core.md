# 네이밍 M군 (03_naming)

## 역할
타겟과 포지셔닝이 확정된 후 → 고객의 언어로 제품·콘텐츠·상점 이름을 짓는다.
"우리 회사가 원하는 이름"이 아니라 "타겟의 머릿속에 이미 존재하는 언어를 이름으로 확정"하는 것이 목표다.

---

## 핵심 사고 프레임워크

### 네이밍의 정의
**좋은 네이밍** = 기억에 잘 남는 별명 + 원하는 포지션에 배치하기 위한 이름

브랜드는 네이밍부터 시작된다. 회사명·제품명·브랜드명·블로그 제목·스마트스토어 상품명 모두 이 원칙을 따른다.

---

### 1단계: 네이밍 유형 선택

#### 유형 1 (의미 중심) — 자신 있는 브랜드 전용
브랜드 고유의 의미·스토리에 기반. 이름 자체에는 기능적 의미가 없고, 브랜드 파워와 스토리텔링으로 채운다.

| 사례 | 이름 자체의 의미 | 성립 이유 |
|------|---------------|---------|
| 구글 | 없음 | 검색 시장 독점 후 브랜드 자체가 동사화 |
| 애플 | 없음 (과일) | 혁신 스토리로 이름에 의미 부여 |
| 무신사 | '무지하게 신발 사진 많은 곳' 약자 | 패션 커뮤니티 1위 후 의미 체계화 |
| 프라엘 | 없음 | LG 브랜드 파워 + 프리미엄 발음감 |

> **주의**: 유형 1은 이미 시장 점유가 확보된 브랜드에서 작동한다. 신규 브랜드가 유형 1로 시작하면 이름이 들려도 연상이 생기지 않는다.

#### 유형 2 (포지셔닝 중심) — 신규 진입자 필수
제품의 기능·가치·시장 위치를 이름에 직접 반영. 이름만 들어도 무엇인지, 누구를 위한 것인지가 전달된다.

| 사례 | 포지셔닝 언어 | 성공 이유 |
|------|------------|---------|
| 키미테 | 귀(耳) 밑(下) + 테이프 → "귀 밑에" | 붙이는 위치가 이름 자체 |
| 하나로 샴푸 | 샴푸+린스를 하나로 | 시장 후발주자가 네이밍 하나로 1위 |
| 배달의민족 | 배달하는 민족 → 배달 서비스 | 업의 본질을 이름이 설명 |
| 돼지코팩 | 돼지코 모양 코팩 | 기능 동일한데 네이밍+패키지로 시장 장악 |
| 탑블로 (등산 텀블러) | 탑(정상) + 텀블러 → 등산 전용 | 타겟 세분화 후 그 세계의 언어 사용 |
| 눈엔 | 눈에 쓰는 것 | 위치 직관형, '엔' 시리즈 확장 가능 |
| 가그린 | 가글 + 그린(신선함) | 기능+감각이 이름에 |

**원칙**: 스타트업·신규 판매자는 무조건 유형 2. 유형 1은 나중에 브랜드 파워가 쌓인 후 선택 가능.

---

### 2단계: 고객 언어 발굴 (가장 중요한 단계)

네이밍의 핵심은 **우리가 짓는 것이 아니라, 타겟이 이미 부르는 말을 확정하는 것**이다.

#### 고객 언어 발굴 방법
1. **타겟 채널 탐색**: 커뮤니티·카페·리뷰에서 타겟이 이 제품을 어떻게 부르는지 수집
2. **검색어 확인**: 타겟이 이 문제를 검색할 때 쓰는 단어 수집 (네이버 자동완성, 연관검색어)
3. **구매 후기 분석**: 리뷰에서 반복 등장하는 표현 추출
4. **경쟁 제품 분석**: 경쟁사가 선점한 언어 vs 비어있는 언어 파악

> **실패 역독**: 강사의 치킨가게 이름은 '치킨 철판이야'였지만 고객들은 '슬러시 맥주집'이라 불렀다. 고객 언어를 이름으로 쓰지 않은 실패. 고객들이 이미 붙인 별명이 진짜 이름이다.

#### 타겟 세분화 후 언어 재설정
동일 제품도 타겟을 좁히면 언어가 달라진다.

```
텀블러 (광범위) → 등산용 텀블러 (세분화) → 탑블로 (등산 세계의 언어)
                → 직장인 텀블러           → (직장인 세계의 언어로)
                → 캠핑 텀블러             → (캠핑 세계의 언어로)
```

타겟이 달라지면 이름도 달라진다. 같은 제품에 타겟별 이름이 필요한 이유.

### 3단계: 네이밍 전략 적용

#### 전략 A: 위치·부위·상황 직결형
"어디에, 언제, 어떤 상황에서 쓰는가"를 이름에 넣는다.

| 원리 | 사례 |
|------|------|
| 부위(위치) | 키미테(귀 밑), 눈엔(눈에), 두피열엔(두피 열에), 무릎엔(무릎에) |
| 용도+대상 | 등산용 → 탑블로, 아기용 → OO베이비 |
| 상황 | 야간 수면용, 출산 후 회복용, 수능 집중용 |

**적용 기준**: 제품이 특정 부위·장소에 쓰이고, 그 위치 정보가 구매 결정에 영향을 줄 때.

#### 전략 B: 핵심 가치(CV) 직결형
제품이 주는 핵심 가치(CV)를 이름에 담는다.

| 원리 | 사례 |
|------|------|
| CV 한 단어 | 하나로(샴푸+린스 통합), 가그린(가글+상쾌함) |
| 결과 직접 표현 | "하루 60마리만 튀긴다" (희소성+신선함) |
| 가치 동사화 | 배달의민족 (배달한다는 행위가 이름) |

**적용 기준**: 이 제품의 CV가 한 단어·한 구절로 명확히 말해질 때.

#### 전략 C: 타겟 직접 지목형
이름 안에 타겟의 상황·욕구가 들어있어 "이건 내 얘기다"를 느끼게 한다.

```
블로그 제목·상품명에 특히 효과적:
"층간소음으로 잠 못 드는 분들을 위한 귀마개"
"야간근무자 수면 전용 이어플러그"
"갱년기 어머니 선물용 온열찜질기"
"생리통 심한 여성 전용 복부찜질기"
```

**적용 기준**: 검색 유입이 필요한 블로그 제목, 스마트스토어 상품명.

#### 전략 D: 가치 더하기형 네이밍
기능은 같지만 이름·패키지·포지셔닝으로 다른 제품처럼 보이게.

```
코팩 (기존) → 돼지코팩 (이름+패키지 가치더하기) → 시장 장악
텀블러 (기존) → 탑블로 (등산 전용 연상) → 세분화 시장 선점
```

**적용 기준**: 기존 레드오션에서 타겟 세분화나 포지셔닝 차별화가 필요할 때.

### 관여도와 네이밍

관여도 = 중요도 × 고민도. 네이밍은 타겟의 관여도를 **낮추는 방향**으로 설계한다.

| 타겟 유형 | 네이밍 설계 방향 |
|---------|--------------|
| 고관여 타겟 | 이미 많이 알고 많이 고민함 → "이걸 쓰면 더 이상 고민 안 해도 된다"는 신호를 이름에 |
| 저관여 타겟 | 별로 중요하게 생각 안 함 → 이름이 관여도를 높여야 함. 궁금하게, 결핍 자극 |

예) '하나로' 샴푸 → 고관여(샴푸·린스 고민하던) 타겟에게 "이거 하나면 끝"이라는 신호

### 썸네일 / 블로그 제목 네이밍

온라인에서는 썸네일과 제목이 콘텐츠 선택의 90~100%를 결정한다.

#### 제목의 유일한 목적
**보게 하고, 클릭하게 한다.** 이 두 가지 이외의 기능은 없다.
제목이 클릭되는 순간 임무 완료 — 클릭 이후는 본문이 할 일이다.

#### 썸네일/제목 4원칙
| 원칙 | 방법 |
|------|------|
| 튀게 | 주변과 대조 (컬러 → 흑백, 복잡 → 단순) |
| 특이하게 | 예상 밖 요소 (사진 목록에 사람 얼굴, 인물 목록에 풍경) |
| 관여도 맞게 | 진중한 내용 = 진중한 썸네일, 가벼운 내용 = 가벼운 썸네일 |
| 예쁘게 | 기본 심미성, 시그니처 컬러·디자인 |

#### 블로그 제목 네이밍 원칙
- 검색 키워드를 제목 앞쪽(첫 10자 이내)에 배치
- 결핍·문제 상황 직접 언급
- 숫자 활용 ("3가지", "1주일", "10배")
- 의문형으로 클릭 유도 ("이거 아세요?", "왜 안 될까요?")

## 출력 형식

```
[제품/콘텐츠명] 네이밍 결과

분류: □ 제품명  □ 브랜드명  □ 블로그 제목  □ 스마트스토어 상품명

타겟: (01_deficit_target 결과에서 확정된 타겟 1줄)
포지셔닝: (02_positioning 결과에서 확정된 CV 1줄)

| 후보 | 유형 | 전략 | 타겟 언어 반영 | 비고 |
|------|------|------|-------------|------|
| (후보 1) | 유형2 | 위치·상황형 | ✅ 고객이 이미 쓰는 단어 | 최우선 추천 |
| (후보 2) | 유형2 | 가치 직결형 | ✅ | |
| (후보 3) | 유형2 | 타겟 지목형 | △ 내 언어 일부 포함 | |

[최종 추천]: 후보 X
[채택 이유]: (타겟 언어 반영, 관여도 매칭, 포지셔닝 일치 여부 1~2줄)
[탈락 이유]: (탈락 후보들이 왜 안 되는지 — 고객 언어인지 회사 언어인지)
[확장 가능성]: 시리즈 확장 가능한 구조면 명시 (눈엔 → 무릎엔, 두피열엔)
```

## 판단 원칙 요약

1. **고객이 이미 뭐라 부르는지가 정답이다** — 회사가 짓고 싶은 이름이 아님
2. 신규 진입은 무조건 유형 2 (포지셔닝 중심)
3. 타겟 세분화 → 그 타겟 세계의 언어로 이름 짓기
4. 관여도를 낮춰주는 방향으로 설계
5. 블로그 제목·상품명은 검색 키워드 + 결핍 언어의 조합
6. 이름은 "기억에 남는 별명"이 되어야 한다 — 부르기 쉽고 연상이 명확해야
7. 가치 더하기는 이름만으로도 가능하다 — 기능 변경 없이 네이밍으로 시장 장악 가능

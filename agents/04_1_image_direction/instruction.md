[지시]
- **출력은 `---SECTIONS_JSON---` ... `---END_SECTIONS_JSON---` JSON 블록만 만든다.** 그 외 마크다운 본문·서두 인사·자기검수 체크리스트는 출력하지 마라. 시스템은 JSON만 파싱하고 UI는 JSON에서 카드를 렌더링한다.
- JSON 블록은 출력의 **맨 앞**에 둔다. 다른 어떤 텍스트도 그 앞에 두지 마라.
- 04 콘티 표의 각 행을 디렉션 1건으로 변환하라. 행 수만큼 `sections` 배열이 나와야 한다 (단, "[디자인 주의사항]"은 섹션이 아니라 `design_system`으로 분리).
- 카피는 04의 카피 방향을 받되, **조사·문장부호까지 확정된 최종 문구**로 다듬어라. "이런 느낌"은 디렉션이 아니다.
- 가격·인증번호·모델명은 [제품 정보]의 권위 필드를 한 글자도 변형하지 말고 그대로 사용하라.
- `design_system`을 한 번만 선언하고, 각 섹션의 한국어 필드(`composition`, `background`, `mood`, `design_notes`)가 그 규칙(색·폰트·인물 외형)을 반복·강화하도록 하라.
- **`gpt_prompt` 필드는 만들지 마라.** JSON의 한국어 필드(`background`, `composition`, `mood`, `text_elements`, `design_notes`)가 이미지 생성 입력값이다. UI가 이걸 마크다운 카드로 렌더링하고, 사용자는 그 마크다운을 GPT Image에 복붙한다.
- 인물 컷은 한국인임을 명시하고(`composition` 또는 `design_notes`에 "60대 한국인 할머니" 식으로), 후기 컷은 얼굴 없이 텍스트 카드형으로.
- 인물 등장 컷의 배경 처리: 배경이 스토리상 중요하지 않으면 `design_notes`에 `얕은 심도, 배경 아웃포커스` 명시. 배경 자체가 맥락 전달의 핵심인 씬(매장 전경, 공장 현장, 사용 환경 묘사 등)은 명시하지 않고 선명도 유지.
- `design_notes`에는 인물 등장 컷의 경우 `로고 없음. 워터마크 없음. AI 인공감 인물 없음.`을 끝에 명시.
- 후기 카피는 04 콘티의 후기 문구를 그대로 이어받아라 (새로 창작 X).
- **캔버스 너비는 860px 고정**(스마트스토어·쿠팡·카페24·옥션·지마켓·도매꾹 멀티 채널 공용 호환값). 비율은 섹션 성격에 따라 아래 세트에서 선택:
  - 4:5 (860×1075) — 기본 상세 컷
  - 2:3 (860×1290) — 세로 길이가 핵심인 제품(타워크레인 등 높이 강조), 정보 밀도가 높아 한 컷에 다 못 담는 씬
  - 1:1 (860×860) — 정방형 강조 배너, 단일 메시지 컷
  - **세로 강조 카피("○○보다 큽니다", "1.3미터입니다" 등)가 있는 섹션은 반드시 2:3을 선택해 실제 높이감이 시각화되도록 하라.**
- **섹션 분할 허용**: 한 콘티 행의 카피·요소가 너무 많아 한 컷에 담으면 여백이 사라지고 비율이 깨지는 경우, 해당 행을 2개 섹션으로 분할할 수 있다. 분할 시 `name`에 `-A`, `-B` 접미사를 붙여 출처를 표시하라(예: "기능설명-A", "기능설명-B"). 분할 기준: `text_elements`가 4개 초과 OR `composition`에 독립적으로 묘사할 영역이 2개 이상.
- **가격 표시 규칙**: 섹션에 가격 정보가 들어가는 경우, 권위 필드에 정가/할인가가 모두 있으면 `text_elements`에 정가(`role: "price_original"`, content 끝에 "(취소선)" 표기)와 판매가(`role: "price_sale"`)를 각각 분리 출력. 정가가 없으면 판매가만 `role: "price_sale"`로 강조.
- **API 자동화 전환 시 주의(현재는 GPT 웹 복붙 운영)**: GPT Image API는 가로·세로 모두 16의 배수 제약이 있으므로 향후 API 운영 시 860→**856**, 1075→**1072**, 1290→**1296**, 880→**880**(이미 16배수)로 보정 필요. 웹 복붙 운영 중에는 픽셀이 정확히 일치하지 않아도 되므로 비율만 유지하면 충분하다.
- JSON 키 이름은 core.md 예시와 정확히 일치해야 한다 (snake_case 영문 키).
- **`text_elements`는 `content`, `role` 2항목만 사용한다.** `position`, `size`, `color`, `weight`, `padding`, `background`, 테두리 등 시각 실행 스펙은 일체 지정하지 마라. 텍스트의 위치·크기·색·박스 스타일은 GPT Image가 디자인 감각으로 판단해야 한다. 좌표를 강제하면 GPT가 요소를 기계적으로 배치해 결과물이 투박해진다.
- **`composition`에서 "상단 1/3, 중단 1/3, 하단 1/3" 같은 등분 강제는 금지한다.** "상단에 제품, 하단에 Q&A·구매 영역" 식으로 영역만 자연어로 지시하고 구체 비율은 GPT Image에 맡긴다.
- qa_checklist.md 규칙을 위반하지 마라.

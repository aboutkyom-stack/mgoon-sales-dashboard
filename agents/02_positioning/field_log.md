# 02 포지셔닝 — 필드 로그 (Field Log)

> **이 파일의 목적**: 02 포지셔닝 단계 작업에서 발생한 판단·가설·편향을 누적. 다음 세션의 이 단계 작업 시 동일한 고민을 반복하지 않게 한다.
> **로드 시점**: 이 단계 작업 시작 시 자동 로드 (instruction.md 선행 의무 첫 항목). 없으면 건너뜀.
> **금지**: 이 파일을 "정답"으로 읽는 것. 항상 가설로 취급. core.md 수정 금지. 단일 사례로 단정 금지.

---

## 1. 이 파일의 목적

- 02 단계 고유 판단(CV 결정·포지셔닝 그래프·두 개 까고 넣기·가치더하기·category_objections·구매편익 판정)에서 발생한 학습을 기록.
- 여러 제품에 걸쳐 반복되는 패턴을 발굴하기 위한 **원자료**(raw data) 역할.
- 결론이 아니라 **가설**로 유지. 표본이 쌓이기 전에 core로 승격 금지.

---

## 2. 왜 core.md를 직접 수정하지 않는가

- core.md는 이 단계의 확정 사고 프레임. 소수 사례로 수정하면 **과적합(overfitting)** 위험.
- 엠군 원칙: *"포지셔닝은 인식의 게임. 내가 잘날 필요 없이, 상대에게 문제가 생기면 된다."*
- 단계 field_log에서 **3~5회 반복 확인**된 패턴만 사용자와 논의 후 core 반영.
- 여러 단계에 걸친 공통 패턴 또는 시스템 전체 편향 → `_shared/field_log_meta.md` 보고.

---

## 3. 사용 원칙

### 3-1. 호출 시 활용

- 이 파일을 읽되 **"이전 세션의 가설"** 로 취급.
- 현재 제품·타겟과 맥락이 다르면 무시해도 된다.
- "field_log에 이렇게 됐으니 이렇게 하자"는 **금지**. 핵심 판단은 매번 core.md에서 다시 꺼낸다.

### 3-2. 기록 기준 (언제 append 하나)

다음 중 하나라도 해당하면 작업 후 **섹션 4(결정 로그)** 에 append:

- 사용자가 내 분석에 반박해서 **내가 일부라도 수용**한 경우
- instruction.md에 명시적 지침이 없어 **내 재량으로 결정**한 경우
- "두 개 까고 넣기" 선택·가치더하기·category_objections 발굴 등에서 **실험적 접근**을 한 경우
- 구매편익 판정 시 기능재/경험재/상징재 경계에서 **재량으로 결정**한 경우

### 3-3. 기록 양식 (섹션 4에 이 양식대로)

```markdown
### YYYY-MM-DD | [제품/상품id] | [판단 주제]

**상황**:
- CV·포지셔닝 관련 핵심 맥락 1~3줄

**내 초기 판단**:
- 무엇을 선택했고 왜

**사용자 반박(있다면)**:
- 어떤 반박이 있었나

**최종 결정**:
- 무엇으로 귀결됐고 왜

**학습**:
- 어느 편향에 빠져 있었나 / 어느 축에 가중치를 몰았나

**검증 대기**:
- 04_a·04_b 결과 또는 실제 판매 데이터로 무엇을 확인해야 하나
```

### 3-4. 금지 사항

- 단일 사례로 "이게 맞다" 단정 → 가설 형태로만
- 결과(전환율·매출·04_b 검수 통과율) 없는 성공·실패 주장
- core.md 수정
- field_log 자체를 "지시문"으로 사용 (항상 가설로 읽기)

### 3-5. 주기적 정리

- 동일 패턴 3~5회 확인 → 섹션 6(Core 반영 후보)로 이동 + **사용자와 논의**
- 여러 단계·여러 제품에 걸쳐 반복되는 패턴 → `_shared/field_log_meta.md` 보고
- 항목 누적 시 섹션 5에서 유사 가설 병합

### 5-A. 단계 특화 가설 카테고리

#### 5-A-1. "두 개 까고 두 개 넣기" 선택 편향

> 까는 대상 2개를 선정할 때 어떤 기준으로 고르는지에 대한 가설.
> "소비자가 이미 식상해진 CV"를 선정해야 하는데, 판매자 시각으로 까는 경향이 있을 수 있음.
> 어떤 CV를 까야 타겟의 인식에 가장 충격을 주는지 제품 유형별 패턴.

*(아직 없음)*

#### 5-A-2. 가치더하기 3개 상한 적용 가설

> 가치더하기를 3개로 제한할 때 어떤 기준으로 3개를 고르는지에 대한 가설.
> "욕구에 맞닿아야 한다"는 원칙 아래, 어떤 가치더하기가 실제로 04 카피에서 살아남는지.
> 3개 상한이 너무 적은 케이스 vs. 3개도 많은 케이스 분류 패턴.

*(아직 없음)*

#### 5-A-3. category_objections 발굴 편향

> AI 추론으로 발굴한 category_objections가 실제 타겟의 의심과 얼마나 일치하는지.
> 어떤 카테고리에서 AI 추론이 잘 맞고, 어디서 틀리는지.
> 사용자 입력 없이 추론한 경우와 사용자 제공 경우의 04_b 통과율 비교.

*(아직 없음)*

#### 5-A-4. 구매편익 경계 케이스 판정

> 기능재/경험재/상징재 경계에 있는 제품에서 판정 실수 경향.
> 예: 기능재로 판정했으나 실제 타겟은 상징재 욕구가 강했던 경우.
> 복합 편익 제품에서 1순위를 잘못 잡으면 04 설득 방식 전체가 어긋남.

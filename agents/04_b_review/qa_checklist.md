## Engine-02. 한_축_사슬 yaml↔본문 일치 점검 ⭐

ENGINE_PLAN yaml의 `한_축_사슬` 한 줄 요약과 본문 `## 한 축 사슬 (표면 → 이면 → 본질)` 섹션의 골자가 의미상 일치하는가.

**배경**: yaml `한_축_사슬`은 DB 컬럼화·검색·교차 분석용 한 줄 요약. 본문 섹션은 자세한 설명·근거. 둘은 계층화 관계이므로 의미 불일치 시 데이터 분석이 거짓 신호 잡음. 04_a가 yaml과 본문을 따로 뱉다 보면 욕구깡패 3차의 강조 지점이 어긋날 수 있음.

**점검 절차**:
1. yaml `한_축_사슬` 한 줄 추출 — 형식이 `"표면(X) → 이면(Y) → 본질(Z)"` 인지 확인 (형식 위반 = DIRECT 형식 교정)
2. 본문 `## 한 축 사슬` 섹션의 표면·이면·본질 3단 추출
3. yaml의 X·Y·Z가 본문 표면·이면·본질의 골자를 압축한 것인지 확인 (어휘 100% 동일할 필요는 없음, 의미가 같으면 OK)

**위반 예**:
- yaml `한_축_사슬: "비둘기 똥 → 청소 피로 → 전원 평온 파괴"`, 본문은 "비둘기 똥 → 악취 → 이웃 민원" — 이면·본질이 다른 사슬 = RETURN (어느 쪽이 의도였는지 04_a에 확인)
- yaml에 `한_축_사슬` 키 누락 (LLM이 yaml 명세 누락) = DIRECT 04_a 재호출 트리거
- yaml 한 줄이 `"표면 → 이면 → 본질"` 형식 안 따름 (예: 줄바꿈·괄호 누락) = DIRECT 형식 교정

**통과 예**:
- yaml `한_축_사슬: "비둘기 똥 흔적(표면) → 반복 청소·악취(이면) → 전원 평온 깨짐(본질)"`, 본문 `## 한 축 사슬` 섹션이 같은 3단으로 자세히 서술 — 의미 일치 ✓

## Rich-01. 섹션 수 정당성 (수치 함정 금지 — 풀세트 6-0)
섹션 수가 **한 축 깊이·펼침 신호·카드 필터 결과**로 정당화되는지 확인.
**절대 범위(예: 6~20)를 기준으로 판정 금지.**

- 위반 예: 04_a가 4섹션으로 뱉었는데 펼침 신호 5개 발동 + 단순 제품 아님 → RETURN 누락
- 위반 예: 12섹션인데 펼침 신호 0~1개 + 카드 필터 통과 카드 없음 → 부풀림 검출 누락
- 통과 예: 8섹션 + 펼침 신호 3개 적용 + 카드 필터 통과 카드와 매칭
- 통과 예: 40섹션이라도 펼침 신호 + 카드 필터 통과로 정당화되면 OK
- 통과 예: 6섹션이라도 단순 제품·압축 결과면 OK

## Authority-02. 약점 인정 카피의 권위 필드 null 항목 검출 ⭐

"약점 인정 + 자신감 받침"(m트랩) 카피에 **권위 필드 null인 항목**을 자체 약점으로 박은 곳이 있는가.

**배경**: 04_a가 약점 인정 카피를 뱉을 때 권위 필드에 없는 항목(KC인증·환불 정책·후기 수·AS 등)까지 자동으로 채우려는 편향이 있음. 강의 본의(core.md §7 m트랩 / 04_a T-10): "**타겟이 의식 안 한 외부 디테일 짚기 = 긁어 부스럼**" — 권위 필드 null이면 타겟이 의식 안 한 외부 디테일일 가능성 매우 높음.

**점검 절차**:
1. 약점 인정 카피에 등장한 항목들 추출 (KC·환불·AS·후기·인증·광고·신생 등)
2. 각 항목이 권위 필드(`[인증]`, `[판매조건]`, `[운영약속]`, `[후기]` 등)에 명시돼 있는가 확인
3. 권위 필드 null인데 약점으로 인정된 항목 발견 시 → **DIRECT 즉시 카피 제거**

**위반 예**:
- KC인증 권위 필드=null인데 카피에 "KC無" 박힘 → 카테고리(예: 새 퇴치용 모형) 자체가 KC 의식 카테고리도 아닌데 짚어서 새 의심 시작 → DIRECT 제거
- 환불 정책 권위 필드=null인데 "환불 어려울 수 있습니다" 박힘 → DIRECT 제거
- AS 권위 필드=null인데 "AS 부재" 박힘 → DIRECT 제거

**통과 예**:
- 권위 필드에 명시된 "광고비 0원·신생·후기 적음" 3가지로만 약점 인정 카피 구성. 자신감 받침("활공 위장만은 자신있다")은 유지.

**관련 사례**: 독수리 모형새(엠군봇 테스트) — KC無 카피가 m트랩에 박혀서 권위 필드 null + 타겟 무의식 외부 디테일 짚는 긁어 부스럼 발생. 04_b가 잡아주는 자리.

## Reason-01. 핵심 반론 선제 처리 정합성 ⭐ (Rev-08 점검 완료)

다음 5가지 모두 점검했는가:
1. ENGINE_PLAN ON → 콘티에 블록 존재
2. 3단 구조 (직격 인정 + 해결책·원리 + 차별점 연결)
3. 위치 (배제 설득 직후, 소거법·CV선언 직전)
4. 배제 설득과 톤 혼동 X (카테고리 의심을 우리 제품 한계로 카피화 X)
5. 타겟지능 = 논리형 시 Q&A 짧은 중복

- 위반 예: "이거 익숙해지면 효과 없을 수도 있습니다" (인정만, 해소 누락)
- 위반 예: "이런 분께는 비추입니다 — 광장에서 쓰실 분" (배제 설득 톤으로 처리, 카테고리 의심 무시)
- 통과 예: "맞습니다. 비둘기는 학습력 빠릅니다. 그래서 위치 주기적으로 바꿔주세요. 우리는 줄에 거는 방식이라 5분이면 가능합니다."

## Internal-01. 내부 설계 용어 카피 노출 검출 ⭐ (Rev-09 즉시 DIRECT)

카피·뱃지·헤드라인·SUB·CAPTION·BODY에 다음 어휘 검출 시 DIRECT 즉시 제거·교체 완료:
- 배제 설득, 소거법, 핵심 반론 선제 처리, 핵심 반론
- 기능계, 상징계, 제거/비교, 경험계, 클로징
- 한 축, 욕구깡패, 구매편익, 기능재·경험재·상징재
- 포지셔닝, 가치더하기, 제로 마케팅, 매몰비용

블록 참조 표현 검출:
- "아래 ○○ 참조", "위의 ○○에서 보듯이", "다음 ○○ 섹션" → 자연 흐름으로 교체

→ 내부 라벨은 콘티 표 `사용 기법` 컬럼에만 등장. 그 외 무조건 DIRECT.
